Languages & Grammars

A language is a set of strings over an alphabet; a grammar defines which strings belong to the language.
Core Definitions
  • Alphabet (Σ): finite set of symbols
  • String: finite sequence of symbols from Σ; empty string = ε
  • Language (L): any subset of Σ* (all possible strings over Σ)
  • Σ*: Kleene closure — all strings including ε
  • Concatenation: xy joins string x and string y

Regular ⊂ Context-Free ⊂ Context-Sensitive ⊂ Recursively Enumerable. Each class is a strict subset of the one above it.
Production Rules

A grammar G = (V, T, S, P) where V = variables, T = terminals, S = start symbol, P = productions.

Type 3 (Regular): A → aB or A → a (right-linear) Type 2 (CFG): A → α (A single variable; α any string) Type 1 (CSG): αAβ → αγβ (|γ| ≥ 1) Type 0: α → β (no restriction)

Derivation
A string w is in L(G) if S =>* w via a sequence of production applications. =>* denotes zero-or-more steps. Leftmost derivation always expands the leftmost variable first.

Finite Automata & Regular Languages

A DFA accepts exactly the class of regular languages; every NFA has an equivalent DFA.
DFA Formal Definition

A DFA is a 5-tuple M = (Q, Σ, δ, q0, F):

  • Q: finite set of states
  • Σ: input alphabet
  • δ: Q × Σ → Q: transition function
  • q0 ∈ Q: start (initial) state
  • F ⊆ Q: set of accept (final) states
Acceptance Criterion
M accepts w = a1 a2 ... an iff r0=q0, ri+1 = δ(ri, ai+1), rn ∈ F
NFA vs DFA

NFA to DFA (Subset Construction)
Each DFA state = a set of NFA states. Worst case: exponential blowup, but both recognise the same class.
Regular Expressions
  • ∅ = empty language; ε = empty string
  • a = language {a} for each a ∈ Σ
  • R1 | R2 = union; R1R2 = concatenation
  • R* = Kleene star (zero or more repetitions)
Pumping Lemma
For any regular language L, there exists p (pumping length) such that any string w ∈ L with |w| ≥ p can be split w = xyz with |xy| ≤ p, |y| ≥ 1, and xyiz ∈ L for all i ≥ 0. Used to prove non-regularity.
DFA Minimisation (Myhill-Nerode)

Two strings x, y are distinguishable w.r.t. L if ∃z s.t. exactly one of xz, yz is in L. The minimum DFA for L has one state per equivalence class of indistinguishable strings.

  • Table-filling algorithm marks distinguishable pairs bottom-up
  • Merge all pairs of unmarked (equivalent) states
  • Result: unique (up to isomorphism) minimal DFA
Example: DFA for strings ending in "ab"
States: q0 (start), q1 (seen 'a'), q2 (seen 'ab', accept). δ(q0,a)=q1, δ(q0,b)=q0, δ(q1,a)=q1, δ(q1,b)=q2, δ(q2,a)=q1, δ(q2,b)=q0.

Pushdown Automata & CFGs

A PDA is a finite automaton augmented with an unbounded stack; it recognises exactly the context-free languages.
PDA Formal Definition

A PDA is a 6-tuple M = (Q, Σ, Γ, δ, q0, F):

  • Γ: stack alphabet
  • δ: Q × Σε × Γε → P(Q × Γε): transition relation
  • Reads input symbol, pops stack symbol, pushes new string, transitions state
Context-Free Grammar (CFG)

G = (V, T, S, P) where every production has form A → α with A ∈ V and α ∈ (V ∪ T)*.

Classic CFG: balanced parentheses S → SS | (S) | ε Palindromes over {a,b}: S → aSa | bSb | a | b | ε
Parse Trees
  • Root = start symbol; leaves = terminals; interior = variables
  • Yield of tree = concatenation of leaves (left to right)
  • Ambiguity: a grammar is ambiguous if some string has two or more distinct parse trees
Ambiguity
Some CFLs are inherently ambiguous — no unambiguous CFG exists for them. Ambiguity causes problems in compilers (expression evaluation order).
Closure Properties of CFLs
  • Closed under: union, concatenation, Kleene star, reversal
  • NOT closed under: intersection, complement
CYK Algorithm
Cocke-Younger-Kasami: O(n3) dynamic programming algorithm to decide if w ∈ L(G) for any CFG G in Chomsky Normal Form.
Normal Forms for CFGs
  • Chomsky Normal Form (CNF): every production is A → BC or A → a. Any CFG can be converted to CNF; required for CYK.
  • Greibach Normal Form (GNF): every production is A → aα where a is a terminal. Useful for constructing PDAs; guarantees each step consumes one input symbol.
CFL Pumping Lemma
For any CFL L, ∃p s.t. any w ∈ L with |w| ≥ p can be split w = uvxyz with |vxy| ≤ p, |vy| ≥ 1, and uvixyiz ∈ L for all i ≥ 0. Proves languages like {anbncn} are not context-free.

Turing Machines

A Turing Machine is the most powerful computational model; defines the boundary of what is algorithmically solvable.
Formal Definition

A TM is a 7-tuple M = (Q, Σ, Γ, δ, q0, qacc, qrej):

  • Q: finite set of states
  • Σ: input alphabet (blank symbol _ ∉ Σ)
  • Γ ⊇ Σ: tape alphabet (includes blank)
  • δ: Q × Γ → Q × Γ × {L, R}: transition function
  • q0: start state; qacc: accept; qrej: reject
Configuration & Computation
Configuration: (q, tape_contents, head_position) Written as: uqv where u = left of head, q = current state, v = right of head Yields: C1 |- C2 (single step) Accept: C0 |-* Cacc (any number of steps)
Halting & Behaviour
  • Accept: enters qacc — computation terminates, input accepted
  • Reject: enters qrej — computation terminates, input rejected
  • Loop: computation never terminates (infinite loop)
Church-Turing Thesis
Every effectively computable function is computable by a Turing Machine. This is a thesis, not a theorem — it cannot be formally proved, but no counterexample has ever been found.
TM Variants (all equivalent)
VariantDescription
Multi-tape TMk independent tapes; simulated by 1-tape TM with O(t2) overhead
Non-deterministic TMMultiple transitions allowed; accepts if any branch accepts
EnumeratorPrints (enumerates) all strings in a language

Recognisers vs Deciders
A TM recognises L if it accepts all w ∈ L (may loop on w ∉ L). A TM decides L if it always halts and accepts iff w ∈ L.
Universal Turing Machine (UTM)

A single TM U that, given encoding <M, w>, simulates M on w. The UTM proves that a stored-program computer is theoretically realisable; U is the conceptual ancestor of modern CPUs.

U(<M, w>) = M(w) Accepts iff M accepts w Rejects iff M rejects w Loops iff M loops on w
Encoding TMs
TMs can be encoded as binary strings (<M>). There are countably many TMs but uncountably many languages, so most languages have no TM — most languages are undecidable.

Decidability

The Halting Problem is undecidable: no TM can determine for all (M, w) pairs whether M halts on w.

The Halting Problem (HP)

ATM = { <M, w> | M is a TM and M accepts w }

Proof by diagonalisation (Turing, 1936): Assume decider H exists for A_TM. Build D: on input <M>: run H on <M, <M>> if H accepts -> D REJECTS if H rejects -> D ACCEPTS Run D on <D>: D accepts <D> iff D rejects <D> -- contradiction!
Undecidable Problems
Halting Problem, emptiness of TM language, equivalence of two TMs, Post Correspondence Problem (PCP), 10th Hilbert Problem. None can be decided by any algorithm.
Reductions

Language A reduces to B (A ≤m B) if there is a computable function f such that w ∈ A ⇔ f(w) ∈ B.

  • If B is decidable and A ≤m B, then A is decidable
  • If A is undecidable and A ≤m B, then B is undecidable
Rice's Theorem
Rice's Theorem
Every non-trivial semantic property of TM-recognisable languages is undecidable. ("Non-trivial" = not true for all TMs, not false for all TMs.)
Semi-Decidable Languages
  • L is semi-decidable (r.e.) iff a TM enumerates L or recognises L
  • L is decidable iff both L and its complement are recognisable
Decidable Problems (examples)
  • ADFA: does DFA M accept string w? — simulate M on w
  • EDFA: is the language of DFA M empty? — reachability from start state
  • EQDFA: do two DFAs recognise the same language?
  • ACFG: does CFG G generate string w? — use CYK
  • ECFG: is the language of CFG G empty? — variable reachability

Complexity Classes — P vs NP

P = problems solvable in polynomial time; NP = problems verifiable in polynomial time. P vs NP is the greatest open problem in CS.
Class P

P = { L | L decided by a DTM in time O(nk) for some k }.

  • Reachability in graphs, sorting, shortest paths, primality testing
  • Closed under complement, union, intersection, concatenation
Class NP

NP = { L | L decided by a NTM in polynomial time } = { L | L has polynomial-time verifier }.

Verifier definition: L ∈ NP iff exists poly-time V such that w ∈ L ⇔ exists certificate c, |c|=O(|w|^k), V(<w,c>) accepts
NP-Complete

L is NP-complete if: (1) L ∈ NP, and (2) every A ∈ NP reduces to L in polynomial time (L is NP-hard).

Cook's Theorem (1971)
SAT (Boolean satisfiability) is NP-complete. SAT was the first problem proven NP-complete; all subsequent NP-completeness proofs reduce from SAT or its variants.
Key NP-Complete Problems
  • SAT / 3-SAT: satisfiability of propositional formulas / 3-CNF
  • CLIQUE: does G have a k-clique?
  • VERTEX-COVER: cover all edges with k vertices?
  • HAMILTONIAN-PATH: path visiting every vertex once
  • TSP: Traveling Salesman Problem (decision version)
  • SUBSET-SUM: subset summing to target t
Polynomial Reduction
A ≤_p B (poly-time mapping reduction): Computable f in poly time s.t. w ∈ A ⇔ f(w) ∈ B If B ∈ P and A ≤_p B then A ∈ P If A is NP-hard and A ≤_p B then B is NP-hard
Practical Implication
If P = NP, every NP problem has efficient (polynomial) solution. Most cryptography would break. Widely believed P ≠ NP, but unproved. Millennium Prize Problem ($1M prize).

NP-Hard vs NP-Complete
NP-hard = at least as hard as every NP problem (need not be in NP itself). NP-complete = NP-hard AND in NP. Halting Problem is NP-hard but not in NP.

Common Exam Pitfall
NFA and DFA recognise the same class (regular languages) — but NFA can be exponentially more concise. Similarly, non-deterministic TMs and deterministic TMs recognise the same class, but time complexity may differ (NTM poly-time ⇔ DTM exp-time, unless P = NP).
Recognisability Test
L is decidable iff L is recognisable AND complement of L is recognisable. If only L is recognisable, L may be undecidable (e.g., ATM).
