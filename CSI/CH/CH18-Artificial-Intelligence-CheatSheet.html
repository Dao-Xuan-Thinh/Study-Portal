AI Fundamentals

AI = the study and design of intelligent agents that perceive their environment and take actions that maximize the chance of success
Definition

Artificial Intelligence aims to build systems that can reason, learn, plan, and act in ways that would require intelligence if performed by humans.

Brief History
  • 1950 — Alan Turing proposes the Turing Test as a criterion for machine intelligence
  • 1956 — Dartmouth Conference coins "Artificial Intelligence" as a field
  • 1974–80 — First AI Winter; funding cuts after unmet expectations
  • 1987–93 — Second AI Winter; expert-system limitations exposed
  • 1997 — Deep Blue defeats Kasparov at chess
  • 2012–now — Deep learning renaissance driven by big data and GPUs
Strong vs Weak AI
TypeDescriptionStatus
Strong AIGeneral intelligence equal to human cognitionTheoretical / not achieved
Weak AISpecialised system designed for a narrow taskCurrent state of the art
Major AI Subfields
  • Machine Learning (ML) — systems that improve from experience
  • Natural Language Processing (NLP) — understanding and generating human language
  • Computer Vision — interpreting visual information from images/video
  • Robotics — designing agents that act in the physical world
  • Expert Systems — encoding domain knowledge in rule-based systems
  • Planning & Search — finding sequences of actions to reach a goal
Interdisciplinary Roots
AI draws from mathematics, cognitive science, linguistics, psychology, neuroscience, and computer engineering — making it one of the broadest fields in CS.

Search Algorithms

Search = systematically exploring a state space to find a goal state or optimal sequence of actions
State Space Representation
  • State: a configuration of the problem at a given point
  • Initial state: where the search begins
  • Goal state(s): desired configurations
  • Actions / operators: transitions between states
  • Path cost: cumulative cost of reaching a state
Uninformed (Blind) Search
AlgorithmStrategyComplete?Optimal?
BFSLevel-by-level (FIFO queue)YesYes (unit cost)
DFSDeepest node first (stack)No (loops)No
Iterative DeepeningDFS with increasing depth limitYesYes (unit cost)
Uniform CostLowest path cost firstYesYes
Iterative Deepening Key Point
Combines BFS's completeness and optimality with DFS's low memory usage — the preferred uninformed strategy for large, unknown search spaces.
Informed (Heuristic) Search
  • Greedy Best-First — expands node with lowest heuristic h(n); fast but not optimal
  • A* Search — uses f(n) = g(n) + h(n); optimal if h is admissible
f(n) = g(n) + h(n) g(n) = cost from start to n h(n) = estimated cost from n to goal
Game Trees & Adversarial Search
  • Minimax algorithm — alternates MAX/MIN levels; explores full game tree
  • Alpha-beta pruning — eliminates branches that cannot affect the result; same outcome as minimax with far fewer nodes visited
Alpha-Beta Efficiency
With perfect move ordering, alpha-beta reduces the branching factor from b to √b, effectively doubling the search depth for the same cost.

Knowledge Representation

Knowledge representation = encoding real-world knowledge in a form a computer can reason about
Propositional Logic

Uses propositions (true/false statements) connected by logical connectives: ¬ (NOT), ∧ (AND), ∨ (OR), → (implies), ↔ (iff).

  • Decidable — every statement is either true or false
  • Limited expressiveness — cannot quantify over objects
Predicate (First-Order) Logic

Extends propositional logic with objects, predicates, and quantifiers (∀, ∃).

∀x Human(x) → Mortal(x) Human(Socrates) ⊢ Mortal(Socrates)
Inference Rules
RuleFormMeaning
Modus PonensP, P→Q ⊢ QFrom P and P implies Q, conclude Q
Modus Tollens¬Q, P→Q ⊢ ¬PContrapositive reasoning
Resolution(A∨B), (¬A∨C) ⊢ B∨CCore of automated provers
Knowledge Base (KB)
  • Stores facts and rules about a domain in a structured form
  • Inference engine applies rules to derive new knowledge
  • Foundation of expert systems and logic programming (Prolog)
Semantic Networks & Frames
  • Semantic network — graph of nodes (concepts) and labelled edges (relations, e.g., is-a, has-a)
  • Frames — structured records with slots and values, inspired by Minsky (1975); precursor to object-oriented programming
Open World vs Closed World
Closed-world assumption (expert systems): anything not known is false. Open-world assumption (the Web/ontologies): unknown facts may be true.

Machine Learning

ML = programs that improve their performance on a task through experience (data), without being explicitly programmed for every case
Supervised Learning

Learns from labelled training data — input/output pairs — to predict outputs for unseen inputs.

  • Classification — output is a discrete category (spam/not-spam, image label)
  • Regression — output is a continuous value (house price, temperature forecast)
  • Algorithms: decision trees, k-NN, SVM, logistic regression, random forests
Unsupervised Learning

Finds hidden structure in unlabelled data.

  • Clustering — groups similar instances (k-means, hierarchical, DBSCAN)
  • Dimensionality reduction — compresses features while preserving structure (PCA, t-SNE, autoencoders)
  • Density estimation — models the underlying data distribution
Reinforcement Learning

An agent takes actions in an environment, receives rewards/penalties, and learns a policy that maximises cumulative reward.

Q(s,a) ← Q(s,a) + α[r + γ·max Q(s',a') − Q(s,a)] Q-learning update rule (Bellman equation)
Model Fit
ProblemTraining ErrorTest ErrorRemedy
OverfittingLowHighRegularisation, more data, dropout
UnderfittingHighHighMore complex model, better features
Good FitLowLow—
Evaluation Protocol
  • Train/Validation/Test split — training tunes weights, validation tunes hyperparameters, test gives final unbiased estimate
  • k-fold cross-validation — partitions data into k subsets; each serves as test set once
Data Leakage Warning
Never use test-set data during training or hyperparameter tuning — it invalidates the evaluation and leads to over-optimistic performance estimates.

Neural Networks

Neural network = layered architecture of interconnected computational units (neurons) inspired by the biological brain
Biological Inspiration
  • Biological neuron: dendrites receive signals → cell body integrates → axon fires output
  • Artificial neuron: weighted inputs → summation → activation function → output
  • Synaptic strength ↔ learned weight parameter
Perceptron (Single Neuron)
output = f(w₁x₁ + w₂x₂ + … + wₙxₙ + b) w = weights, x = inputs, b = bias

A single perceptron can only solve linearly separable problems (e.g., AND, OR — but not XOR).

Multilayer Networks (MLP)
  • Input layer — receives raw feature values
  • Hidden layer(s) — learn intermediate representations
  • Output layer — produces final prediction
Activation Functions
FunctionFormulaRangeUse Case
Sigmoid1/(1+e⁻ˣ)(0, 1)Binary classification output
tanh(eˣ−e⁻ˣ)/(eˣ+e⁻ˣ)(−1, 1)Hidden layers
ReLUmax(0, x)[0, ∞)Default for deep nets
Softmaxeˣⁱ / Σeˣʲ(0,1), Σ=1Multi-class output
Backpropagation
  1. Forward pass — compute predictions and loss
  2. Compute loss gradient w.r.t. output layer weights
  3. Propagate gradients backward through layers (chain rule)
  4. Update each weight: w ← w − η·∂L/∂w
Vanishing Gradient
Sigmoid/tanh gradients saturate near 0 or 1, causing early layers to learn very slowly in deep networks. ReLU and batch normalisation mitigate this.
Deep Learning Overview
  • CNN (Convolutional Neural Network) — spatial feature extraction; dominant for images
  • RNN / LSTM — sequential data; handles temporal dependencies
  • Transformer — attention mechanism; state-of-the-art for NLP (BERT, GPT)
  • Requires large datasets, GPU compute, and careful regularisation

AI Applications

AI transforms raw data into actionable intelligence across language, vision, autonomous systems, and decision support
Natural Language Processing (NLP)
  • Tokenisation — splitting text into tokens for processing
  • Parsing — analysing grammatical structure (syntax trees)
  • Sentiment analysis — classifying text as positive/neutral/negative
  • Machine translation — seq2seq models (encoder-decoder, Transformer)
  • Question answering — retrieving or generating answers from context
Computer Vision
  • Image classification — assigning a category label to an entire image
  • Object detection — localising and classifying multiple objects (YOLO, Faster R-CNN)
  • CNNs — hierarchical feature extraction via learned filters
  • Semantic segmentation — pixel-level classification
  • Facial recognition — identity verification from face images
Expert Systems
  • Encode domain expertise as IF-THEN rules in a knowledge base
  • Inference engine applies forward chaining (data → conclusion) or backward chaining (goal → supporting facts)
  • Examples: MYCIN (medical diagnosis), DENDRAL (chemical analysis)
Robotics
  • Perception (sensors), planning (path/motion), and actuation (effectors)
  • SLAM — Simultaneous Localisation and Mapping
  • Industrial robots, surgical robots, autonomous vehicles
Ethical Concerns
Bias & Fairness
Models trained on biased data amplify existing societal biases in hiring, lending, and criminal justice applications.
Privacy & Surveillance
Facial recognition and behavioural tracking raise serious privacy concerns — regulation is an active policy debate.
Explainability (XAI)
Deep learning models are largely black boxes; explainable AI research aims to make decisions transparent and auditable.

Quick Review
Search Strategy Comparison
BFS: complete, optimal (unit cost), O(bd) memory.
DFS: incomplete (loops), not optimal, O(bm) memory.
A*: complete, optimal (admissible h), memory-intensive.
Alpha-beta: same result as minimax, O(bd/2) vs O(bd).
Machine Learning Workflow
1. Collect & clean data  →  2. Feature engineering  →  3. Choose model  →  4. Train on training set  →  5. Tune on validation set  →  6. Evaluate on test set  →  7. Deploy & monitor.
Neural Network Design Checklist
✔ Choose architecture matching data type (CNN for images, Transformer for text).
✔ Normalise inputs.
✔ Use ReLU activations in hidden layers, softmax/sigmoid at output.
✔ Apply dropout and batch normalisation to reduce overfitting.
✔ Monitor training vs. validation loss curves for early stopping.
Common AI Pitfalls
• Confusing correlation with causation in ML models.
• Leaking test-set information into training.
• Ignoring class imbalance (use F1, precision, recall — not just accuracy).
• Deploying without considering distributional shift.
