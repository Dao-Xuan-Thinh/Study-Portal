§4.1 — XOR, NAND, NOR

XOR — Exclusive OR

Result is 1 when the two input bits differ; 0 when they are the same.

10110011 XOR 00001111 = 10111100

Key XOR Properties
  • A XOR A = 0 — same value cancels
  • A XOR 0 = A — zero is identity
  • A XOR 1 = NOT A — ones flip the bit
  • Uses: toggle bits, parity checking, simple encryption, swap trick
NAND & NOR — Universal Gates

NAND = NOT(AND): any logic function can be built from NAND alone.

NOR = NOT(OR): also a universal gate.

ℹ XOR Identity Trick
A XOR B XOR B = A — XOR-ing with the same value twice cancels out. Used in RAID-5 parity, simple encryption, and the no-temp swap: a^=b; b^=a; a^=b

§4.2 — Shift Operations

Shift operations move all bits left or right — equivalent to fast multiplication/division by powers of 2
Shift Types
TypeSymbolFillEffectUse
Logical Left<<0s on right×2ⁿUnsigned multiply
Logical Right>>0s on left÷2ⁿ unsignedUnsigned divide
Arithmetic Left<<0s on right×2ⁿSigned multiply
Arithmetic Right>>Sign bit copies÷2ⁿ signedSigned divide
Rotate Left<<<MSB → LSBCircularCryptography
Rotate Right>>>LSB → MSBCircularCryptography
Example — Unsigned: 01110110 (118₁₀)

Left shift 1: 11101100 (236₁₀ = 118×2)

Right shift 1: 00111011 (59₁₀ = 118÷2)

Example — Signed: 11110000 (−16 in 8-bit 2's complement)

Arith right shift 1: 11111000 (−8 = −16÷2) ← sign preserved

Logic right shift 1: 01111000 (120) ← sign lost, wrong!

n << k = n × 2ᵏ
n >> k = n ÷ 2ᵏ (arithmetic for signed, logical for unsigned)
⚠ Overflow Warning
Overflow on left shift: if a significant bit shifts out, the result is wrong. Always use arithmetic right shift (not logical) for signed division.

Carry pattern from right: 0,1,1,0,0,0,1,0

Overflow Detection

Unsigned overflow: final carry-out = 1 → result doesn't fit in n bits.

Signed overflow: carry into MSB ≠ carry out of MSB.

⚠ Signed Overflow
Adding two positives yields a negative, or adding two negatives yields a positive → signed overflow has occurred!

§4.3 — Subtraction via Two's Complement

Computers compute A − B as A + (−B) using two's complement — no separate subtract circuit needed
Steps
  1. Find two's complement of B: flip all bits, then add 1
  2. Add A + (−B) using normal binary addition
  3. Interpret MSB as sign (1 = negative in 2's complement)
Example: 54 − 77 in 8-bit

77 = 01001101
flip → 10110010
+1 → 10110011 (= −77)
──────────────────
54 = 00110110
+ (−77) = 10110011
──────────
          11101001

MSB=1 → negative. Decode: flip 11101001 = 00010110, +1 = 00010111 = 23 → result = −23 ✓

Example: 77 − 54 in 8-bit

54 = 00110110
flip → 11001001
+1 → 11001010 (= −54)
──────────────────
77 = 01001101
+ (−54) = 11001010
──────────
          00010111 = 23 ✓

Carry-out bit is discarded for signed arithmetic.

💡 Hardware Insight
The ALU has an inverter on one input — to subtract, it inverts B and sets carry-in to 1 (which adds the +1 for two's complement negation). One adder circuit handles both add and subtract.

§4.3 — Multiplication & Division

Multiplication — Shift-and-Add Method

For each bit of the multiplier (right to left): if bit = 1, add the appropriately shifted multiplicand.

Example: 1011 (11) × 0101 (5)

bit0=1: add 0001011 (shift 0)
bit1=0: skip
bit2=1: add 0101100 (shift 2)
bit3=0: skip
──────────
Sum: 0001011 + 0101100
     = 0110111 = 55 ✓

Division
  • Hardware uses restoring or non-restoring algorithms
  • Division by 2ⁿ = arithmetic right shift n positions
Quick Powers-of-2 Shortcuts
n × 2ᵏ = n << k (left shift k bits)
n ÷ 2ᵏ = n >> k (arithmetic right shift k bits)

📝 Note
Real CPUs use Booth's algorithm for efficient signed multiplication and hardware dividers — but the shift-and-add principle is the conceptual foundation.

Common Bitwise Tricks
  • Check even/odd: n & 1 == 0 → even
  • Check power of 2: n & (n-1) == 0 (and n > 0)
  • Swap without temp: a ^= b; b ^= a; a ^= b
  • Clear lowest set bit: n & (n-1)
  • Isolate lowest set bit: n & (-n)
  • Set bit k: n | (1 << k)
  • Clear bit k: n & ~(1 << k)
  • Toggle bit k: n ^ (1 << k)
  • Test bit k: (n >> k) & 1
💡 Where Bit Manipulation Is Used
Bit manipulation is used in embedded systems, graphics engines, cryptography, network packet parsing, and anywhere performance is critical.
