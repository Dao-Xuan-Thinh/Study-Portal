Language Evolution

Evolution: machine → assembly → high-level → 4GL → 5GL
Machine Language (1GL)

Machine language is the native binary instruction set of the CPU. All programs ultimately execute as machine language.

  • Instructions encoded as 0s and 1s
  • Completely hardware-dependent and non-portable
  • Example: 1011 0000 0110 0001 (MOV AL, 61h on x86)
Assembly Language (2GL)

Assembly language replaces binary opcodes with human-readable mnemonics.

  • 1:1 correspondence with machine instructions
  • Requires an assembler to translate to machine code
  • Still processor-specific; used for drivers and embedded systems
  • Example: MOV AX, BX / ADD CX, 5
High-Level Languages (3GL)

Abstract the hardware; one statement may generate many machine instructions.

  • Portable across hardware platforms
  • Examples: C, C++, Java, Python, Pascal, FORTRAN, COBOL
  • Require a compiler or interpreter
Generational Table
GenNameExamples
1GLMachine LanguageBinary opcodes
2GLAssembly Languagex86 ASM, ARM ASM

Portability Trend
As generation increases, programmer productivity rises and hardware dependence decreases. 3GL+ languages are largely portable; 1GL/2GL are architecture-specific.

Translation

Source code → (assembler/compiler/interpreter) → executable
Assembler

Translates assembly language into machine code. Produces an object file. Near 1:1 translation; extremely fast.

Compiler vs Interpreter
AspectCompilerInterpreter
TranslationWhole program at onceLine by line at runtime
OutputStandalone executableNo separate executable
SpeedFaster executionSlower (overhead per line)
Error detectBefore executionAt runtime (per line)
PortabilityPlatform-specific binarySource runs anywhere
ExamplesC, C++, Rust, GoPython, Ruby, PHP
Linker

Combines multiple object files and library references into a single executable. Resolves external symbol references between modules.

  • Static linking: libraries embedded in the executable
  • Dynamic linking: shared libraries (.dll / .so) loaded at runtime
Loader

Part of the OS that copies the executable from disk into memory and sets up the runtime environment before transferring control to the program's entry point.

JIT Compilation
Just-In-Time (JIT)
Hybrid approach: bytecode is compiled to native machine code at runtime, just before first execution. Used by Java (JVM) and .NET (CLR). Combines portability of interpreters with speed approaching native compilers.
Java Pipeline
Java source → javac (compiler) → .class bytecode → JVM (interpreter + JIT) → native execution
Full Build Pipeline Summary
  1. Write source code in high-level language (.c, .java, .cpp)
  2. Preprocessor — expands macros and #include directives (C/C++)
  3. Compiler — translates source to assembly or object code; reports syntax/type errors
  4. Assembler — converts assembly to machine code object file (.o / .obj)
  5. Linker — merges object files + libraries → executable (.exe / a.out)
  6. Loader — OS maps executable into memory; sets up stack, heap, registers
  7. CPU begins executing at program entry point (main())

Programming Paradigms

A paradigm is a fundamental style or approach to programming
Imperative vs Declarative
StyleFocusExamples
ImperativeHow to do it — explicit stepsC, Pascal, Assembly
DeclarativeWhat to do — describe the resultSQL, HTML, Prolog
Procedural (Structured)

Programs are sequences of statements organized into procedures/functions. Control flows top-down.

  • Languages: C, Pascal, FORTRAN, COBOL
  • Emphasizes: subroutines, sequence, selection, iteration
  • State changed via variable assignment
Object-Oriented (OOP)

Organizes code around classes and objects that model real-world entities.

  • Languages: Java, C++, Python, C#, Ruby
  • Core pillars: encapsulation, inheritance, polymorphism, abstraction
Functional

Treats computation as evaluation of mathematical functions. Avoids mutable state and side effects.

  • Languages: Haskell, Lisp, Erlang, Scala, F#
  • Features: first-class functions, immutability, recursion, higher-order functions
  • No loops — iteration via recursion or map/fold
Logic (Declarative)

Programs are sets of logical facts and rules; the engine infers solutions.

  • Language: Prolog
  • Programmer states what is true; runtime finds how
  • Used in AI, natural language processing, expert systems
Note
Most modern languages are multi-paradigm: Python, C++, and Scala support procedural, OOP, and functional styles simultaneously.

OOP Concepts

OOP models software as interacting objects encapsulating state and behavior
Class vs Object
TermDefinition
ClassBlueprint/template defining fields (data) and methods (behavior)
ObjectRuntime instance of a class; has its own state in memory
Dog fido = new Dog("Fido", 3); // Java: class Dog, object fido
Encapsulation

Encapsulation bundles data and the methods that operate on it into a single unit, hiding internal implementation.

  • Access modifiers: private, protected, public
  • Getter/setter methods expose state safely
  • Prevents unintended external modification
Inheritance

Inheritance allows a subclass to extend a superclass, reusing and specializing its behavior.

  • Java: class Cat extends Animal { ... }
  • C++: class Cat : public Animal { ... };
  • Single (Java) vs multiple inheritance (C++)
  • Promotes code reuse; establishes IS-A relationships
Polymorphism

Polymorphism — "many forms" — same method call behaves differently based on the actual object type.

  • Compile-time (static): method overloading — same name, different parameters
  • Runtime (dynamic): method overriding via virtual dispatch
// Java: runtime polymorphism Animal a = new Dog(); a.speak(); // calls Dog.speak(), not Animal.speak()
Abstraction

Abstraction exposes only essential features, hiding complexity.

  • Achieved via abstract classes and interfaces
  • Java: interface Shape { double area(); }
  • Enables programming to an interface, not an implementation

Language Components

Every high-level language shares core structural components
Primitive Data Types
TypeDescriptionExample
intWhole numbersint x = 42;
floatSingle-precision decimalfloat pi = 3.14f;
doubleDouble-precision decimaldouble e = 2.718;
charSingle character (ASCII/Unicode)char c = 'A';
boolTrue or falsebool flag = true;
Composite / Derived Types
  • Array: contiguous block of same-type elements; indexed from 0 — int arr[5];
  • String: sequence of characters (array of char or object)
  • Pointer (C/C++): variable that stores a memory address — int *p = &x;
  • Struct/Record: group of related fields under one name
Variables & Constants
  • Variable: named memory location whose value can change — int count = 0;
  • Constant: fixed value that cannot be reassigned — const int MAX = 100;
  • Variables have: name, type, value, scope, lifetime
Expressions & Statements
  • Expression: combination of values and operators that evaluates to a value — x + y * 2
  • Statement: complete instruction that performs an action — x = x + 1;
  • Assignment: stores a value in a variable — a = b + c;
Type Safety
Strongly typed languages (Java, Haskell) enforce type rules at compile time. Weakly typed languages (JavaScript, C) allow implicit type coercion, which can cause subtle bugs.
Pointer Caution
Pointers give direct memory access (power & performance) but are a major source of bugs: null dereferences, dangling pointers, buffer overflows. Java and Python avoid raw pointers.

Functions / Methods

A function encapsulates a reusable computation. In OOP, functions inside a class are called methods.

  • Signature: returnType name(paramType param, ...)
  • Promotes DRY principle (Don't Repeat Yourself)
  • May return a value, or void if none
Parameter Passing
MethodBehaviorLanguages
Pass by ValueCopy of argument; original unchangedC (default), Java (primitives)
Pass by ReferenceAlias to original; changes affect callerC++ (&), C# (ref)
Pass by Object RefReference copy; object state can changeJava (objects), Python
Scope
  • Local scope: variable visible only within its block { }
  • Global scope: variable accessible throughout the program
  • Block scope: limited to the enclosing braces (C, Java)
  • Lexical (static) scope: determined at compile time (most languages)
Recursion

A function that calls itself to solve a smaller sub-problem. Requires a base case to terminate.

int factorial(int n) { if (n == 0) return 1; // base case return n * factorial(n - 1); // recursive case }
Recursion vs Iteration
Recursion is elegant for tree/graph traversal and divide-and-conquer algorithms. Iteration is generally more memory-efficient (avoids stack frames). Tail recursion can be optimized to iteration by some compilers.
Exception Handling

Most high-level languages provide structured error handling to separate normal logic from error logic.

  • try block — code that might throw an error
  • catch block — handles a specific exception type
  • finally block — always executes (cleanup); Java, C#, Python
  • C++ uses throw / catch; Python uses raise / except
Standard Library & Modules
  • Every language ships a standard library (I/O, math, strings, collections)
  • Code is organized into modules / packages / namespaces
  • Imported on demand: import java.util.ArrayList; / import math
  • Promotes separation of concerns and code reuse across projects
  • Third-party libraries extend functionality (npm, pip, Maven, NuGet)

Quick Review — Chapter 9 Essentials
Language Generations
1GL = machine binary → 2GL = assembly mnemonics → 3GL = high-level (C, Java) → 4GL = domain-specific (SQL) → 5GL = AI/logic (Prolog). Each generation increases programmer productivity and abstraction.
Compiler vs Interpreter Trade-off
Compilers produce fast executables but require a separate build step and are platform-specific. Interpreters offer immediate execution and easier debugging but run slower. JIT tries to capture both benefits.
Translation Pipeline
Source code → Compiler/Assembler → Object files → Linker → Executable → Loader → Process in memory → CPU execution
Common Pitfalls
  • Missing base case in recursion → infinite loop / stack overflow
  • Confusing pass-by-value and pass-by-reference → unexpected side effects
  • Variable shadowing (inner scope hides outer) → logic bugs
  • Treating inheritance as the only reuse mechanism — prefer composition over inheritance when IS-A relationship is unclear
