Compression Concepts

Compression = removing redundancy from data without (or with limited) loss of information
Lossless vs Lossy
PropertyLosslessLossy
ReconstructionExact / perfectApproximate
Reversible?YesNo
Typical ratio2:1 – 8:110:1 – 100:1
Use casesText, executables, medicalAudio, video, photos
ExamplesZIP, PNG, FLAC, LZWJPEG, MP3, MPEG, AAC
Compression Ratio & Redundancy
Ratio = size_original / size_compressed Redundancy = 1 − (1 / Ratio) Example: 1000 B → 250 B ⟹ Ratio = 4:1, Redundancy = 75%
Types of Redundancy
  • Spatial (data) redundancy — repeated patterns within a single image or file
  • Temporal redundancy — similar content across consecutive video frames
  • Coding redundancy — fixed-length codes for symbols of unequal probability
  • Perceptual redundancy — details imperceptible to human senses (hearing, vision)
Entropy — Theoretical Limit
H = −Σ p_i · log₂(p_i) [bits per symbol] Lower entropy → more predictable data → more compressible Max entropy (uniform dist.) → least compressible
Real-World Applications
File archiving (ZIP/gzip), image formats (PNG, JPEG, GIF, WebP), video streaming (H.264, H.265, AV1), audio distribution (MP3, AAC, FLAC), network packet compression, database storage.
Shannon's Source Coding Theorem
It is impossible to compress data below its entropy rate on average. Huffman and arithmetic coding approach this limit for lossless methods.

Run-Length Encoding (RLE)

Replace runs of identical symbols with (count, symbol) pairs — O(n) time and space
Encoding Algorithm
  1. Read input symbol by symbol; initialise count = 1
  2. If next symbol equals current symbol: count++
  3. Else: emit (count, symbol) pair; reset count = 1; update current symbol
  4. On end of input: emit final (count, symbol) pair
Worked Example — Text
Input: A A A B B C C C C D D Output: 3A 2B 4C 2D Saving: 11 symbols → 8 values (4 pairs × 2 fields)
Binary Image Example (Fax Line)

Each scan line encodes alternating runs of white (0) then black (1), always starting with white:

Pixels: 0 0 0 0 1 1 0 1 1 1 1 0 Runs: 4-white, 2-black, 1-white, 4-black, 1-white Encoded: [4, 2, 1, 4, 1] (5 values vs. 12 pixels)
Best vs Worst Cases
CaseInput (n symbols)OutputEffect
BestAAAAAAAA (8)8A → 2 fields4× reduction
WorstABCDEFGH (8)1A1B1C1D1E1F1G1H2× expansion!
TypicalMixed runsModerate savings1.5–5× ratio
Real-World Uses
Group 3/4 fax (ITU T.4 / T.6), Windows BMP (BI_RLE8/BI_RLE4 modes), PCX graphics, TIFF optional mode, PostScript, early game sprites.
Expansion Risk
RLE expands data when runs are short. Practical implementations use an escape byte or flag bit to mark encoded pairs vs. literal sequences, avoiding worst-case expansion.

Huffman Coding

Assign shorter bit-strings to more-frequent symbols — provably optimal among prefix codes
Building the Huffman Tree — Step by Step
  1. Count frequency of every distinct symbol; create one leaf node per symbol
  2. Insert all nodes into a min-priority queue sorted by frequency
  3. Dequeue the two nodes with the lowest frequencies
  4. Create an internal node whose frequency = sum of the two children
  5. Enqueue the new internal node back into the priority queue
  6. Repeat steps 3–5 until exactly one node remains — this is the root
  7. Assign codes: traverse left edge → append 0, right edge → append 1
Worked Example — Input "AABCDDDD"
Symbol freqs: A=2 B=1 C=1 D=4 Step 1: combine lowest B(1) + C(1) → BC(2) Step 2: combine BC(2) + A(2) → BCA(4) Step 3: combine BCA(4) + D(4) → root(8) Codes: D → 0 A → 10 B → 110 C → 111
SymbolFreqCodeCode lenBits used
D4014 × 1 = 4
A21022 × 2 = 4
B111031 × 3 = 3
C111131 × 3 = 3

Total: 14 bits vs. 16 bits with fixed 2-bit codes. Average code length = 14 / 8 = 1.75 bits/symbol.

Decoding Example
Bitstream: 0 10 110 111 → D A B C (read left-to-right, match codes greedily)
Prefix-Free (Prefix Code) Property
No codeword is a prefix of another. This guarantees unambiguous decoding: traverse the tree bit-by-bit and emit a symbol whenever a leaf is reached.
Adaptive Huffman
Updates frequency counts and rebuilds the tree on-the-fly during encoding. Single-pass; no pre-scan needed — but more complex to implement.

LZW Compression

Dictionary grows automatically during encoding — no pre-scan and no transmitted dictionary

LZW initialises its dictionary with all single-character entries, then extends it with multi-character strings as they are encountered in the input stream.

Encoding Algorithm
  1. Initialise dictionary with every single-character string (entries 0 … 255 for ASCII)
  2. Set working string W = "" (empty)
  3. Read next input character K; form candidate W + K
  4. If W + K exists in dictionary → set W = W + K and go to step 3
  5. Else → output code for W; add W + K to dictionary; set W = K; go to step 3
  6. End of input: output code for current W
Encoding Trace — "AABABABA"
Init: A=0, B=1 W=A; K=A → AA∉dict → out 0, add AA=2, W=A W=A; K=B → AB∉dict → out 0, add AB=3, W=B W=B; K=A → BA∉dict → out 1, add BA=4, W=A W=A; K=B → AB∈dict(3) → W=AB W=AB;K=A → ABA∉dict → out 3, add ABA=5, W=A W=A; K=B → AB∈dict(3) → W=AB W=AB;K=A → ABA∈dict(5) → W=ABA [end of input] Final out: 5 Compressed codes: [0, 0, 1, 3, 5] (5 codes vs. 8 chars)
Decoding
Self-Synchronising Decoder
The decoder reconstructs the exact same dictionary from the code stream alone. No dictionary needs to be sent with the compressed file.
Applications
  • GIF — 8-bit palette images; uses 9–12-bit LZW codes; Unisys patent expired 2003/2004
  • TIFF — LZW is one of several optional compression modes
  • PDF — LZWDecode filter for compressing stream objects
  • Unix compress — classic .Z files; 16-bit LZW variant
Dictionary Overflow
Fixed-size dictionaries eventually fill up. Strategies: freeze dictionary (stop adding), flush and restart, or use a sliding window (LZ77 basis of gzip/deflate).

Lossy Compression — JPEG & MPEG

Irreversible
Once lossy compression is applied, the original data cannot be recovered. Data is permanently discarded to achieve high compression ratios.
JPEG Encoding Pipeline
  1. Color space conversion: RGB → YCbCr; chroma channels subsampled 4:2:0 (human eyes less sensitive to color than luminance)
  2. Block partition: image divided into non-overlapping 8×8 pixel blocks
  3. DCT (Discrete Cosine Transform): each block mapped to 64 frequency coefficients; DC = average, AC = detail
  4. Quantization: each coefficient divided by a value from a quality-dependent quantization table, then rounded — this is the lossy step that discards high-frequency detail
  5. Zigzag scan: reorder 8×8 coefficient matrix into 1-D sequence; low-freq first, trailing zeros grouped at end
  6. Entropy coding: RLE for runs of AC zeros + Huffman coding of remaining non-zero values
JPEG Quality vs. Size

MPEG — Motion Video Compression
Frame types: I-frame (Intra-coded) — full JPEG-like snapshot; self-contained P-frame (Predicted) — encodes difference from nearest prior I/P frame B-frame (Bi-directional) — differences from prior AND future I/P frames Typical GOP: I B B P B B P B B I (Group of Pictures)
  • Motion estimation: finds best-matching block in reference frame; only residual error coded
  • Temporal redundancy: consecutive frames share ≫90% pixel content — P/B frames are tiny
  • B-frames require future reference → encoding order ≠ display order
  • Seeking requires decoding the entire preceding GOP to reach a P or B frame
MPEG Standards Timeline
MPEG-1 (VCD, 1993) → MPEG-2 (DVD / broadcast, 1995) → MPEG-4 (internet, 1998) → H.264/AVC (2003) → H.265/HEVC (2013) → AV1 (2018).

Audio & Video Compression Standards

Audio codecs exploit psychoacoustic masking to discard inaudible data
PCM — Uncompressed Baseline
CD Audio: 44,100 Hz × 16 bits × 2 ch = 1,411,200 bps ≈ 1.4 Mbps → ~10 MB/minute per channel
MP3 — MPEG-1 Audio Layer III
  1. Sub-band filter bank: split audio into 32 equal-width frequency sub-bands
  2. Psychoacoustic model: for each band compute masking threshold
  3. Bit allocation: assign more bits to audible, less-masked bands; zero bits to fully masked bands
  4. MDCT: Modified DCT applied per sub-band for finer frequency resolution (18 coefficients per band)
  5. Huffman encoding: lossless stage compresses the quantized MDCT coefficients
Psychoacoustic Masking Effects
Simultaneous masking: a loud tone masks quieter tones at nearby frequencies at the same instant.
Temporal (pre/post) masking: loud sound masks quieter sounds up to ~200 ms before and ~500 ms after it occurs.
AAC — Advanced Audio Coding (MPEG-4)
  • Successor to MP3; same bit rate → noticeably higher perceptual quality
  • Larger transform window (1024 vs. 576 MDCT coefficients) → better frequency resolution
  • Supports up to 48 full-bandwidth channels; used in YouTube, iTunes, iOS, Android
  • Variants: AAC-LC (low-complexity), HE-AAC (spectral band replication for very low bitrates)

Key Terms & Quick Review

Key Terms
TermDefinition
Lossless compression Reduces file size without discarding any data; exact reconstruction guaranteed
Lossy compression Achieves higher ratios by allowing controlled, irreversible information loss
Compression ratio Ratio of original size to compressed size; e.g., 4:1 means 75% space saved
Entropy Theoretical minimum average bits per symbol; measures information content of source
RLE Run-Length Encoding — lossless; effective only when data contains long repeated runs
Huffman coding Optimal variable-length code: frequent symbols get shorter codes; uses prefix tree
LZW Dictionary-based lossless method; dictionary built on-the-fly; no pre-transmission needed
DCT Frequency-domain transform used in JPEG to separate perceptually important low-frequency data
JPEG Lossy still-image format; pipeline: color convert → 8×8 DCT → quantize → entropy code
MPEG Family of lossy video standards; uses intra (I), predicted (P), bi-directional (B) frames
MP3 Lossy audio codec; sub-band coding + psychoacoustic model + MDCT + Huffman
Quantization Divides DCT coefficients by a quality table; rounds result — discards fine detail irreversibly
Prefix code Variable-length code where no codeword is a prefix of any other; enables unique decodability
Dictionary Table mapping patterns to codes; central to LZW (dynamic) and Huffman (static per file)
I / P / B frames MPEG frame types: intra (full), predicted (1 reference), bi-directional (2 references)
Psychoacoustic masking Perceptual property exploited by MP3/AAC to remove frequencies inaudible to human listeners
When to Choose Each Method
Use Lossless When…
Data must be reproduced exactly: source code, executables, medical/legal images, financial records, text files, program data. Use: ZIP, gzip, PNG, FLAC, LZW, Huffman.
Use Lossy When…
Perceptual quality is sufficient and file size matters: web photographs, music distribution, video streaming, video conferencing. Use: JPEG, MP3, AAC, MPEG, H.264/H.265.
Huffman vs. LZW — Key Differences
Huffman: requires two passes (count frequencies, then encode); frequency table must accompany the file or be pre-agreed. Optimal for known symbol distributions.
LZW: single-pass; dictionary is self-embedded — decoder rebuilds it from the code stream. Better for text with long repeated substrings.
JPEG vs. PNG — Format Choice Guide
JPEG: lossy; excellent for photographs and natural images; small files; avoid for sharp-edged graphics, text, or images requiring exact pixel values.
PNG: lossless (LZ77 + Huffman); larger files; exact pixel reconstruction; supports full transparency; ideal for logos, icons, screenshots.
Compression in the FCS4e Context
Forouzan covers lossless methods (RLE, Huffman, LZW) as foundational algorithms and lossy methods (JPEG, MPEG, MP3) as applied system examples. The chapter bridges data representation (CH14) and security (CH16), where compressed data is often then encrypted.

Common Exam Traps
(1) RLE expands data (up to 2×) when no repeated runs exist — always consider the worst case.
(2) LZW never shrinks its dictionary once an entry is added — it only grows (or resets).
(3) Huffman requires two passes; LZW needs one pass.
(4) JPEG quantization is the only lossy step — all other JPEG stages (DCT, zigzag, Huffman) are mathematically lossless.
(5) B-frames in MPEG depend on future frames → transmission order differs from display order.
