§9.1 — The Decimal System

The decimal system uses 10 digits (0–9) and has a base / radix of 10. Every digit is multiplied by 10 raised to a power equal to its position.

MSD vs LSD
Most Significant Digit = leftmost (highest value). Least Significant Digit = rightmost (lowest value).
Position Rule
Each position is weighted 10× the position to its right and 1/10 of the position to its left. Position i is weighted 10i.

§9.2 — Positional Number Systems

A positional number system with radix r represents any number as a string of digits where digit at position i carries weight rⁱ.

General Form
(...a₃a₂a₁a₀ . a₋₁a₋₂a₋₃...)ᵣ
General Value Formula (Eq 9.2)
X = Σᵢ (aᵢ × rⁱ) where: 0 ≤ aᵢ < r (digit range) · = radix point (between a₀ and a₋₁)
The Radix Point

The radix point (decimal point in base 10) separates integer digits (positive powers) from fractional digits (negative powers).

Example — Base 7 Position Weights

Key Insight
Decimal (base 10) is just one special case. The formula Σ(aᵢ × rⁱ) works for ANY radix. Each digit must be in range 0 to r−1.

§9.3 — The Binary System

Binary uses only 2 digits (0 and 1), so it has base/radix = 2. Each binary digit is called a bit.

Key Terminology
  • Bit — single binary digit (0 or 1)
  • Nibble — 4 bits (one hex digit)
  • Byte — 8 bits (two hex digits)
  • MSB — Most Significant Bit (leftmost)
  • LSB — Least Significant Bit (rightmost)
Binary Formula (Eq 9.3)
Y = Σᵢ (bᵢ × 2ⁱ) where bᵢ ∈ {0,1}

§9.4 — Converting Between Binary and Decimal

Binary → Decimal (Integer)

Multiply each binary digit by its positional power of 2 and sum all results.

Method: Σ (bit × 2ⁱ)
Value = bₙ×2ⁿ + ... + b₁×2¹ + b₀×2⁰
Example 1 — Convert 10110₂ to decimal
Bit positions (right to left = pos 0,1,2,...)
1 0 1 1 0
↑ ↑ ↑ ↑ ↑
2⁴ 2³ 2² 2¹ 2⁰
16 8 4 2 1
= (1×16)+(0×8)+(1×4)+(1×2)+(0×1)
= 16+0+4+2+0 = 22₁₀
Example 2 — Convert 11001₂ to decimal
1 1 0 0 1
2⁴ 2³ 2² 2¹ 2⁰
= (1×16)+(1×8)+(0×4)+(0×2)+(1×1)
= 16+8+0+0+1 = 25₁₀
Example 3 — Convert 101010₂ to decimal
1 0 1 0 1 0
2⁵ 2⁴ 2³ 2² 2¹ 2⁰
= 32+0+8+0+2+0
= 42₁₀
Example 4 — Convert 11111111₂ to decimal
1 1 1 1 1 1 1 1
128+64+32+16+8+4+2+1
= 255₁₀ (= 2⁸ - 1)
Shortcut
An n-bit number of all 1s = 2ⁿ − 1. So 8 bits of all 1s = 256−1 = 255.

Decimal → Binary (Integer)

Use repeated division by 2. Remainders collected in reverse order give the binary digits from LSB to MSB.

Algorithm
  1. Divide decimal number N by 2
  2. Record the remainder (0 or 1) — this is the next binary digit (LSB first)
  3. Replace N with the quotient
  4. Repeat until quotient = 0
  5. Read remainders bottom to top (reverse order) for result
Example 1 — Convert 11₁₀ to binary (Fig 9.1a)
Division Quotient Remainder
11 ÷ 2 → 5 R 1 ← LSB (bit 0)
5 ÷ 2 → 2 R 1 ← bit 1
2 ÷ 2 → 1 R 0 ← bit 2
1 ÷ 2 → 0 R 1 ← MSB (bit 3)
Read bottom→top: 1011₂ = 11₁₀ ✓
Example 2 — Convert 21₁₀ to binary (Fig 9.1b)
Division Quotient Remainder
21 ÷ 2 → 10 R 1 ← LSB
10 ÷ 2 → 5 R 0
5 ÷ 2 → 2 R 1
2 ÷ 2 → 1 R 0
1 ÷ 2 → 0 R 1 ← MSB
Read bottom→top: 10101₂ = 21₁₀ ✓
Example 3 — Convert 100₁₀ to binary
100÷2→50 R0, 50÷2→25 R0, 25÷2→12 R1
12÷2→6 R0, 6÷2→3 R0, 3÷2→1 R1, 1÷2→0 R1
1100100₂ = 100₁₀ ✓
Example 4 — Convert 255₁₀ to binary
255÷2=127 R1, 127÷2=63 R1, 63÷2=31 R1
31÷2=15 R1, 15÷2=7 R1, 7÷2=3 R1
3÷2=1 R1, 1÷2=0 R1
11111111₂ = 255₁₀ ✓
Why It Works
N = 2×N₁ + R₀; N₁ = 2×N₂ + R₁; ... Substituting gives N = (Rₙ×2ⁿ)+...+(R₁×2¹)+R₀ — exactly the binary representation.

Decimal Fraction → Binary

Use repeated multiplication by 2. The integer part of each product gives the binary digit; the fractional part is used in the next step.

Algorithm
  1. Multiply fractional part F by 2
  2. The integer part of product (0 or 1) is the next binary digit — MSB first (b₋₁, b₋₂, ...)
  3. Keep only the fractional part of the product as the new F
  4. Repeat until F = 0 or desired precision is reached
  5. Read digits top to bottom (MSB to LSB)
Example 1 — Convert 0.25₁₀ to binary (Fig 9.2b — Exact)
Multiply Product Integer(bit) Fraction
0.25 × 2 = 0.50 → 0 (b₋₁) frac=0.50
0.50 × 2 = 1.00 → 1 (b₋₂) frac=0.00
0.25₁₀ = 0.01₂ (exact)
Example 2 — Convert 0.81₁₀ to binary (Fig 9.2a — Approx)
Multiply Product Integer(bit)
0.81 × 2 = 1.62 → 1 (b₋₁)
0.62 × 2 = 1.24 → 1 (b₋₂)
0.24 × 2 = 0.48 → 0 (b₋₃)
0.48 × 2 = 0.96 → 0 (b₋₄)
0.96 × 2 = 1.92 → 1 (b₋₅)
0.92 × 2 = 1.84 → 1 (b₋₆)
0.81₁₀ ≈ 0.110011₂ (approx, non-terminating)
Example 3 — Convert 0.75₁₀ to binary
0.75 × 2 = 1.50 → 1
0.50 × 2 = 1.00 → 1
0.75₁₀ = 0.11₂ (exact)
Non-Terminating Fractions
Some decimal fractions cannot be represented exactly in binary (e.g., 0.1₁₀). The algorithm is halted after the desired number of bits. This is a source of floating-point rounding errors in computers.
Example 4 — Full Number 34.75₁₀
Integer: 34₁₀ = 100010₂
Frac: 0.75₁₀ = 0.11₂
34.75₁₀ = 100010.11₂

Hex Value Formula (Eq 9.4)
Z = Σᵢ (hᵢ × 16ⁱ) where 0 ≤ hᵢ ≤ F (0–15)
Why Hex? — 3 Reasons (Stallings)
  • More compact than binary notation
  • Computer binary data occupy multiples of 4 bits — one hex digit per nibble
  • Extremely easy to convert between binary and hex
Worked Example — 2C₁₆ to decimal
2C₁₆ = (2×16¹) + (C×16⁰)
= (2×16) + (12×1)
= 32 + 12 = 44₁₀

Binary ↔ Hexadecimal

Group binary digits into nibbles of 4 (from the radix point outward). Each nibble directly maps to one hex digit.

Binary → Hex: Group into 4-bit nibbles
  1. Split binary string into groups of 4 bits from right (pad with 0s on left if needed)
  2. Replace each group with the corresponding hex digit
Example 1 — 110111100001₂ to hex (from §9.5)
1101 | 1110 | 0001
D | E | 1
= DE1₁₆
Example 2 — 1001.1111₂ to hex
1001 . 1111
9 . F
= 9.F₁₆
Hex → Binary: Expand each digit to 4 bits
  1. Replace each hex digit with its 4-bit binary equivalent
  2. Concatenate all groups (remove leading zeros if desired)
Example 3 — A64₁₆ to binary
A 6 4
1010 0110 0100
= 101001100100₂
Example 4 — 1F.C₁₆ to binary
1 F . C
0001 1111.1100
= 00011111.1100₂
Mental Conversion
Experienced programmers can mentally convert between binary and hex visually. The 4-bit grouping is the key — practice until the hex digit table is memorized.

Hex ↔ Decimal Conversion

Convert hex to decimal by expanding positional values (base 16). Convert decimal to hex using repeated division by 16.

Hex → Decimal: Expand positional values
Example: 9F₁₆
= (9×16¹) + (F×16⁰)
= (9×16) + (15×1)
= 144 + 15 = 159₁₀
Example: D52₁₆
= (D×16²)+(5×16¹)+(2×16⁰)
= (13×256)+(5×16)+(2×1)
= 3328+80+2 = 3410₁₀
Example: ABCD₁₆

= 40960 + 2816 + 192 + 13
= 43981₁₀
Decimal → Hex: Repeated division by 16
Example: 2560₁₀ to hex
2560 ÷ 16 = 160 R 0 ← LSD

Read bottom→top: A00₁₆
Example: 255₁₀ to hex

Hex Rollover at 16
Octal rolls over at 8 (7→10₈). Hex rolls over at 16 (F→10₁₆). Binary rolls over at 2 (1→10₂). Compare: 15₁₀=F₁₆=17₈=1111₂.
Powers of 2 Milestones
2⁷=128, 2⁸=256, 2¹⁰=1024 (1K), 2¹⁶=65536 (64K), 2²⁰=1048576 (1M), 2³²≈4.3 billion.
Octal Base-8 Rollover
In octal, digits go 0–7 only. After 7 comes 10 (octal), not 8. So decimal 8 = octal 10, and decimal 64 = octal 100.

Base-r → Base-rⁿ Shortcut
Group digits n at a time. r=2→r²=4: group 2 bits per base-4 digit r=2→r³=8: group 3 bits per octal digit r=2→r⁴=16: group 4 bits per hex digit r=3→r²=9: group 2 base-3 digits per base-9 digit
General Rule
Converting from base r to base rⁿ: group n digits of the original base at a time. This works because (rⁿ)ᵏ = rⁿᵏ.

Hexadecimal Addition

Add hex digit by digit like decimal, but carry when the sum exceeds F (15) instead of 9.

Rule
If sum of column ≥ 16: write (sum−16) as hex, carry 1
Example 1 — Simple (no carry)
3 A 5₁₆
+ 2 4 2₁₆
─────────
5+2=7, A+4=E, 3+2=5
= 5 E 7₁₆

+ 2 7 D₁₆ (= 637₁₀)

8+7+1 = 16 → 16-16=0, carry 1
1+2+1 = 4, no carry

Hexadecimal Subtraction

Subtract column by column. If the digit being subtracted is larger, borrow 16 from the next higher column.

Rule
If top digit < bottom: borrow 1 from left column, add 16 to current digit
Example 1 — Simple (no borrow)
9 F 4₁₆ (= 2548₁₀)
- 3 2 1₁₆ (= 801₁₀)
─────────
4-1=3, F-2=D, 9-3=6
= 6 D 3₁₆ (= 1747₁₀ ✓)
Example 2 — With borrow
C 3 0₁₆ (= 3120₁₀)
- 4 8 F₁₆ (= 1167₁₀)
─────────
0-F: borrow → 16+0-15=1, borrow from 3
3-1(borrow)-8: borrow → 16+2-8=10=A, borrow from C
C-1(borrow)-4 = 7
= 7 A 1₁₆ (= 1953₁₀ ✓)
Verification Tip
Always Verify
After hex arithmetic, convert operands and result to decimal to verify. 3120−1167=1953=7A1₁₆ ✓. This cross-check catches digit/carry errors.
