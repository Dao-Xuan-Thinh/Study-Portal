§5.1 Memory Cell Fundamentals

Core Properties of Every Semiconductor Cell
  • Exhibits two stable states representing binary 1 and 0
  • Writable — can be set to a desired state (at least once)
  • Readable — state can be sensed without destruction (except DRAM read)
Cell Signal Terminals
  • SELECT — activates the cell for read or write
  • CONTROL — indicates read or write mode
  • DATA IN / SENSE — carries data in (write) or state out (read)
DRAM Cell — Analog Capacitor Storage
  • One transistor + one capacitor per bit
  • Write: high/low voltage on bit line → charge stored in capacitor (1 or 0)
  • Read: address line activates transistor → capacitor charge fed to sense amplifier (destructive — capacitor discharged, must be restored)
  • Capacitor leaks charge → must be periodically refreshed
  • DRAM is essentially an analog device — threshold distinguishes 1 from 0
SRAM Cell — Digital Flip-Flop
  • Six transistors (T1–T6) cross-connected in bistable flip-flop configuration
  • Logic state 1: C1 high, C2 low; T1 & T4 off, T2 & T3 on
  • Logic state 0: C1 low, C2 high; T1 & T4 on, T2 & T3 off
  • T5 and T6 controlled by address line to enable read/write
  • No refresh needed — stable as long as power is applied
  • SRAM is a digital device — more power, faster, larger cell

Key Rule of Thumb
DRAM → main memory (cheap + dense). SRAM → cache (fast). Both are volatile — power loss = data loss.
DRAM 16-Mbit Chip (4M × 4) Structure
  • 4 square arrays of 2048 × 2048 cells
  • Only 11 address pins (A0–A10) — address is multiplexed (row then column)
  • RAS = Row Address Strobe; CAS = Column Address Strobe
  • WE = Write Enable; OE = Output Enable
  • Multiplexing doubles memory size for each extra address pin (quadruples with square arrays)

"Random-Access Memory" (RAM) is a misnomer — all types listed above support random access. The true distinguishing features of RAM are: fast read + write via electrical signals and volatility.

Module & Chip Organization

1-Bit-Per-Chip Organization

If a chip provides 1 bit per word, you need at least as many chips as bits per word.

Example: 256K × 8-bit Memory
Use 8 chips of 256K × 1-bit each. 18 address lines feed all chips simultaneously. Each chip contributes one bit to the 8-bit output word.
Scaling to Larger Memories
  • To build 1M × 8: use 4 columns of 8 chips (each column = 256K × 8)
  • Lower 18 address bits → all 32 chips
  • Upper 2 address bits → group-select logic → activates CE on one column
DRAM Address Multiplexing
  • 22-bit address for 2048×2048 array uses only 11 pins — sends row addr then col addr
  • RAS latches row address; CAS latches column address
  • Each new address pin added → memory size quadruples (2× rows × 2× cols)
DRAM Refresh
  • Refresh counter steps through all row values sequentially
  • Each row: RAS activated → cells in row read and written back (restores charge)
  • Chip effectively disabled during refresh cycles

Interleaved Memory

Concept

Main memory composed of multiple independent DRAM banks. Each bank can service a memory request independently.

K banks → service K requests simultaneously Throughput improvement factor ≈ K (for sequential/block accesses)
How It Works
  • Consecutive memory words stored in different banks (interleaved addressing)
  • When processor requests a block, successive words come from successive banks in parallel
  • While one bank completes a transfer, the next bank begins its access cycle
  • Dramatically increases effective memory bandwidth for block transfers
Benefit
Interleaving hides individual DRAM latency by overlapping accesses across banks. Particularly effective for sequential memory access patterns (instruction fetch, DMA transfers).
Limitation
  • If multiple accesses target same bank → no parallelism (bank conflict)
  • Non-sequential access patterns see reduced benefit

§5.2 Error Types & Detection

Categories of Errors
Hard Failure (Permanent)
Physical defect — cell is stuck at 0 or 1, or fluctuates erratically. Caused by environmental abuse, manufacturing defects, or wear. Cannot be fixed by error correction alone.
Soft Error (Transient)
Random, non-destructive event that alters cell content. Hardware is undamaged. Caused by power supply fluctuations or alpha particles (from radioactive decay in packaging materials — distressingly common).
Error-Correcting Code (ECC) Process
  1. On write: compute K-bit code from M data bits using function f; store M + K bits total
  2. On read: recompute K-bit code from M data bits just read
  3. XOR fetched code with recomputed code → produce syndrome word
  4. Syndrome = 0: no error detected → send data out
  5. Syndrome = single bit set: error in a check bit → no data correction needed
  6. Syndrome > 1 bit set: syndrome value = position of erroneous data bit → invert to correct
Simple Parity Bit
  • Single extra bit — set so total 1s in word is even (even parity) or odd
  • Detects single-bit errors only — cannot locate or correct the error
  • Cannot detect even-numbered multi-bit errors (they cancel out)

Hamming SEC Code — Single-Error Correcting (8-bit Data Word Example)

2^r ≥ M + r + 1 Where: r = # check bits, M = # data bits For M=8: r=3 → 2³-1=7 < 8+3 ✗ r=4 → 2⁴-1=15 ≥ 8+4 ✓ ∴ 4 check bits needed for 8 data bits

Check bit positions = powers of 2 (1, 2, 4, 8). Data bits occupy all other positions.

Check Bit Computation (XOR)
C1 = D1⊕D2⊕D4⊕D5⊕D7 (covers odd positions) C2 = D1⊕D3⊕D4⊕D6⊕D7 (covers pos w/ bit1 set) C4 = D2⊕D3⊕D4⊕D8 (covers pos w/ bit2 set) C8 = D5⊕D6⊕D7⊕D8 (covers pos w/ bit3 set) Rule: Ci covers all positions n where bit i of n = 1 e.g., C1 covers positions 1,3,5,7,9,11 (all odd)
Worked Example — Data Word: 00111001
Input: D1..D8 = 0 0 1 1 1 0 0 1 (D1=rightmost) C1 = 1⊕0⊕1⊕1⊕0 = 1 C2 = 1⊕0⊕1⊕1⊕0 = 1 C4 = 0⊕0⊕1⊕0 = 1 C8 = 1⊕1⊕0⊕0 = 0 Stored: 0 0 1 1 0 0 1 1 1 1 1 1 (12 bits)
Error Detection: Data Bit 3 Flipped (0→1)

SEC-DED Concept
Add one extra parity bit covering the entire word. If 2 errors occur, syndrome points to wrong position, but extra parity reveals the discrepancy → detected but not corrected. IBM 30xx: 8-bit SEC-DED per 64 data bits (~12% overhead).

§5.3 Synchronous DRAM (SDRAM)

Asynchronous DRAM Problem

Traditional DRAM is asynchronous — processor presents address, waits for access time delay. CPU must stall (insert wait states) during this delay, degrading performance.

SDRAM Solution
  • Exchanges data synchronized to external clock at full processor/memory bus speed
  • Processor issues instruction + address; SDRAM responds after a set number of clock cycles
  • CPU can perform other work while SDRAM processes — no forced stalling
  • Data is latched by the SDRAM on command receipt
Burst Mode (Key Feature)
  • After first access, consecutive data bits clocked out rapidly without re-sending address
  • Eliminates row/column precharge time for sequential accesses
  • Burst length configurable via mode register (1, 2, 4, 8, full page)
  • CAS latency (delay from read command to first data) also configurable
SDRAM Additional Features
  • Multiple internal banks — improves on-chip parallelism
  • Mode register — customizes burst length and CAS latency at runtime
  • Best for large sequential block transfers (word processors, multimedia, spreadsheets)
  • 256-Mb SDRAM: 4 banks × (8192 × 512 × 16-bit arrays)
Read Timing Example
Burst length=4, CAS latency=2: COMMAND issued at T0 → data appears at T2, T3, T4, T5 (4 consecutive words without re-addressing).

DDR SDRAM Generations — Double Data Rate

How DDR Achieves Higher Throughput (3 Mechanisms)
  1. Double-edge clocking — transfers data on BOTH rising AND falling clock edges (doubles effective rate vs. SDRAM)
  2. Higher bus clock frequency — increased over plain SDRAM
  3. Prefetch buffer — on-chip buffer pre-positions bits for the I/O bus; allows slow DRAM core to feed fast I/O interface
Prefetch Buffer Concept

DRAM core is much slower than I/O interface. Prefetch buffer bridges this gap by:

  • Fetching multiple words from core in parallel
  • Serializing them out through a MUX to the I/O interface
  • DDR1: 2N prefetch → I/O 2× faster than core
  • DDR2: 4N → 4× speedup; DDR3: 8N → 8× speedup
  • DDR4: stays at 8N but adds bank groups for more parallelism
DDR4 Bank Groups
  • Separate bank groups allow column cycles to complete independently
  • Two 8N prefetches can operate in parallel across groups
  • Effectively mimics a 16N prefetch without increasing burst length minimum
  • Up to 4 bank groups supported in DDR4

Core Speed Insight
The DRAM array core speed has not changed significantly across generations (still ~100–266 MHz). Performance gains come entirely from larger prefetch buffers, higher I/O clock, and architectural innovations (bank groups). Each generation = lower voltage + higher effective bandwidth.
Downside of Larger Prefetch
Larger prefetch dictates minimum burst length. DDR3's 8N prefetch makes burst-4 inefficient. DDR4 solved this with bank groups instead of increasing prefetch to 16N.

§5.4 Flash Memory — Operation

Floating-Gate Transistor Structure
  • Normal transistor: control gate + source + drain + P-substrate
  • Flash cell adds a floating gate — insulated by thin oxide layer, sandwiched between control gate and channel
Bit States
Binary 1 — Floating Gate Empty
No electrons trapped on floating gate → transistor operates normally (not obstructed). Cell defaults to 1 state.
Binary 0 — Electrons Trapped
Large voltage applied across oxide layer → electrons tunnel through (Fowler-Nordheim tunneling) and are trapped on floating gate → transistor blocked → cell reads as 0. Electrons remain even with power off (persistent).
Read / Erase
  • Read: external circuitry tests if transistor conducts → 1 if yes, 0 if no
  • Erase: large voltage in opposite direction → removes electrons from floating gate → restores binary 1
  • Erase granularity: entire chip (EPROM-like) or blocks (flash advantage)

Why "Flash"?
Named because an entire section of memory cells is erased in a single action — a "flash." One transistor per bit provides EPROM-like density; electrical erase provides EEPROM-like convenience at block granularity.

NOR vs NAND Flash Memory

Circuit Structure
NOR Flash — Parallel Cell Connection
Cells connected in parallel to bit lines. Each cell individually accessible via its word line. If any cell in a column is on → bit line goes low (like NOR gate behavior). Supports random byte-level access.
NAND Flash — Series Cell Connection
16 or 32 transistors connected in series between bit line and ground. Bit line goes low only if all transistors in the column are on (like NAND gate behavior). Must read/write in pages/blocks — no random byte access.

Wear Leveling

Flash cells degrade with each Program/Erase (P/E) cycle. Without management, frequently written blocks wear out while others remain fresh.

Wear Leveling Algorithm
Flash controller distributes writes evenly across all blocks — even static (cold) data is periodically moved to free up well-worn blocks. Extends overall device lifetime significantly.
Other Flash Limitations
  • No in-place overwrite — must erase entire block before rewriting; old data copied elsewhere first
  • Write amplification — actual writes to flash > logical writes due to block management
  • Read disturb — reading adjacent cells can inadvertently program them over time
  • NAND flash requires Error Correcting Codes (ECC) — controllers always include ECC logic
Persistence
Flash memory is nonvolatile — retains data when power is removed. This distinguishes it from DRAM/SRAM and enables use as secondary storage substitute.

§5.5 STT-RAM (Spin-Transfer Torque RAM)

Technology Basis

New type of magnetic RAM (MRAM) using Magnetic Tunneling Junction (MTJ): thin tunneling dielectric sandwiched between two ferromagnetic layers.

MTJ Structure
  • Reference (pinned) layer — magnetization fixed permanently
  • Free layer — magnetization can be flipped by write current
  • Parallel alignment → low resistance → binary 1
  • Anti-parallel alignment → high resistance → binary 0
Write Mechanism
  • 1st-gen MRAM: current-induced magnetic field flips free layer (large current, poor scaling)
  • STT-RAM: spin-polarized current directly flips free layer magnetization — required current ∝ cell area → better scaling

PCRAM — Phase-Change RAM

Technology Basis

Based on chalcogenide alloy material — same class used in optical storage (CDs/DVDs). Most mature of the new NVM technologies.

Storage Mechanism — Resistance States
Crystalline State → Low Resistance → Binary 0
SET operation: electrical pulse heats material above crystallization temperature → crystallizes → low resistance state.
Amorphous State → High Resistance → Binary 1
RESET operation: larger current pulse melts material then abruptly cut off → rapid quenching → frozen amorphous (disordered) state → high resistance.
Key Properties
PropertyPCRAM
VolatilityNonvolatile

Historical Note
Chalcogenide alloys have long been used in CD/DVD rewritable optical discs where laser heat changes phase. PCRAM applies same phase-change principle electronically with no moving parts.

ReRAM (Resistive RAM) & Memory Hierarchy

ReRAM Technology
  • Also known as RRAM
  • Stores data as resistance values rather than charge
  • Applying electric current to material changes its resistance (forms/breaks conductive filament through metal oxide)
  • Low resistance (filament formed) = binary 0; High resistance (filament broken) = binary 1
  • Low voltage operation; endurance far superior to flash; smaller cells (in theory)
ReRAM vs Flash
  • Lower operating voltage than flash
  • Better endurance than flash memory
  • Smaller cell size potential
  • Suitable for: main memory AND secondary storage replacement

Status (Stallings 10e)
STT-RAM, PCRAM, and ReRAM are all in volume production but remain in specialty applications. NAND Flash still dominates. These emerging memories are expected to gain mainstream adoption in coming years as scaling becomes harder for conventional technologies.

Key Terms Glossary

Memory Types
  • DRAM — Dynamic RAM; capacitor-based; requires refresh; main memory
  • SRAM — Static RAM; flip-flop based; no refresh; cache memory
  • ROM — Read-Only Memory; mask-programmed at fabrication
  • PROM — Programmable ROM; one-time electrically writable
  • EPROM — UV-erasable; rewritable; chip-level erase
  • EEPROM — Electrically erasable byte-by-byte; flexible but slow write
  • Flash Memory — Block-erase NVM; 1 transistor/bit; fast erase
  • NOR Flash — Parallel cells; random access; code execution (XIP)
  • NAND Flash — Series cells; block access; high density; SSDs
  • SDRAM — Synchronous DRAM; clock-synchronized; burst mode
  • DDR DRAM — Double Data Rate; transfers on both clock edges
Newer NVM
  • STT-RAM — Spin-Transfer Torque RAM; magnetic; fast; nonvolatile
  • MRAM — Magnetic RAM; parent category of STT-RAM
  • PCRAM — Phase-Change RAM; chalcogenide alloy; amorphous/crystalline
  • ReRAM / RRAM — Resistive RAM; filament formation in metal oxide
Error Correction
  • Hard failure — Permanent cell defect; stuck at 0/1
  • Soft error — Transient bit flip; no hardware damage; from alpha particles or power issues
  • ECC — Error Correcting Code; K check bits for M data bits
  • Parity bit — Single detection bit; detects but cannot correct
  • Hamming code — SEC code; uses XOR-based check bits at power-of-2 positions
  • Syndrome word — XOR of fetched and recomputed check bits; indicates error position
  • SEC — Single-Error Correcting code
  • SEC-DED — Single-Error Correcting, Double-Error Detecting; SEC + 1 extra parity bit
Organization & Addressing
  • RAS — Row Address Strobe; latches row address into DRAM
  • CAS — Column Address Strobe; latches column address into DRAM
  • Interleaved memory — Multiple banks; K banks → K parallel requests
  • Bank group (DDR4) — Independent subsets of banks; extra parallelism
  • Prefetch buffer — On-chip SDRAM buffer; pre-positions data for I/O bus
  • Burst mode — Sequential data transfer without re-addressing after first access
  • Volatile memory — Loses data without power (SRAM, DRAM)
  • Nonvolatile memory — Retains data without power (ROM, Flash, NVM)

Critical Formulas & Numbers

Hamming SEC check bits: 2^r ≥ M + r + 1 M=8 → r=4 (4 check bits, 12-bit stored word) M=16 → r=5 (5 check bits, 21-bit stored word) M=32 → r=6 (6 check bits, 38-bit stored word) SEC-DED requires r+1 check bits
DRAM address pins: only ⌈log₂(rows)⌉ pins needed (22-bit addr → 11 pins via RAS/CAS multiplexing) Adding 1 address pin → 4× memory size (doubles rows × doubles columns)
Interleaved memory: K banks → up to K× bandwidth (for sequential block access patterns)
DDR effective rate = 2 × clock frequency (both edges) DDR1: 200–400 Mbps | DDR2: 400–1066 Mbps DDR3: 800–2133 Mbps | DDR4: 2133–4266 Mbps
Hamming bit position rule: Ci covers all positions n where bit-i of n = 1 Position n is checked by: Ci where i divides n (e.g., pos 7=0111 → checked by C1, C2, C4)

Common Exam Pitfall
DRAM read is destructive — capacitor discharges during read and must be recharged. This is separate from the periodic refresh requirement. SRAM reads are non-destructive.
