Computer Architecture (ISA)

Architecture = attributes visible to programmer; direct impact on logical execution of a program

ISA (Instruction Set Architecture) is often used interchangeably with "computer architecture."

ISA Defines
  • Instruction formats and opcodes
  • Registers and their usage
  • Instruction and data memory layout
  • Effect of executed instructions on registers and memory
  • Algorithm for controlling instruction execution
Architectural Attribute Examples
  • Instruction set (which operations exist)
  • Number of bits for each data type (integers, characters, floats)
  • I/O mechanisms
  • Techniques for addressing memory
  • Whether a MULTIPLY instruction exists → architectural decision
IBM System/370 Example
Introduced 1970; same architecture has survived to today as IBM mainframe product line. Multiple generations of hardware, all running the same instruction set.

Computer Organization — The HOW

Critical Distinction
Architecture = WHAT the system does (programmer-visible).
Organization = HOW it does it (hardware implementation details, transparent to programmer).
Organizational Attribute Examples
  • Control signals between components
  • Interfaces between computer and peripherals
  • Memory technology used (DRAM vs SRAM)
  • Whether multiply uses a dedicated unit or repeated ADDs
  • Clock speeds and timing
Classic Example — Multiply Instruction

Microcomputers
In microcomputers, architecture and organization are tightly coupled — technology changes affect both simultaneously. RISC (Chapter 15) is a prominent example of interplay between the two.

Structure and Function — §1.2

Hierarchical system: set of interrelated subsystems, each hierarchical in structure, down to elementary components
Two Design Concepts at Every Level
  • Structure: the way components are interrelated
  • Function: operation of each individual component as part of the structure
Hierarchical Design Approach
  • Designer deals with one level at a time
  • Each level: set of components + their interrelationships
  • Behavior at each level depends only on abstracted characterization of next lower level
  • Top-down approach: most clear and effective for description [WEIN75]
Book's Approach
This textbook describes computers top-down: starting at major components → successively lower levels of hierarchy. Structure and function examined together at each level.

Four Basic Computer Functions

Function 1 — Data Processing

Data may take wide variety of forms; only a few fundamental methods/types of data processing exist. The computer must process data in response to program instructions.

Function 2 — Data Storage
  • Short-term: temporarily holds pieces of data being worked on at any given moment
  • Long-term: files of data stored for subsequent retrieval and update
Function 3 — Data Movement
  • I/O (Input–Output): data received from or delivered to a directly connected device (peripheral)
  • Data Communications: data moved over longer distances to/from a remote device
Function 4 — Control

Control unit manages the computer's resources and orchestrates performance of functional parts in response to instructions.

Stallings Quote [SIEW82]
"There is remarkably little shaping of computer structure to fit the function to be performed… all the functional specialization occurs at the time of programming and not at the time of design."

Multicore Computer — Key Definitions

Consensus Definitions (Industry-wide)
TermDefinition
CPUFetches and executes instructions; consists of ALU, control unit, and registers. In single-processor system = "the processor"
CoreIndividual processing unit on a processor chip; may be equivalent to a CPU; other specialized units (e.g., vector units) also called cores
ProcessorPhysical silicon chip containing one or more cores. Interprets and executes instructions. Multicore processor = multiple cores on one chip
Cache Memory Hierarchy
  • Smaller and faster than main memory; speeds up access by storing likely-needed data
  • L1 cache: closest to core; smallest and fastest
  • L2 cache: second level; larger, slower than L1
  • L3 cache: shared by all cores; largest, slowest of cache levels
  • Rule: Level n is smaller and faster than level n+1
IBM zEnterprise EC12
2.75 billion transistors; 6 cores; 2 L3 cache areas shared by all cores; includes ISU, IFU, IDU, LSU, XU, FXU, BFU, DFU, RU, COP, I-cache, L2 control.

IAS Computer — First Stored-Program Computer

Stored-program concept: instructions stored in memory alongside data — programs can be treated as data and manipulated
Historical Background
  • 1945: von Neumann's EDVAC proposal — first publication of stored-program concept
  • Alan Turing developed the idea concurrently
  • 1946: Design of IAS computer began at Princeton Institute for Advanced Studies
  • 1952: IAS computer completed — prototype of all subsequent general-purpose computers
  • EDVAC = Electronic Discrete Variable Computer
IAS Hardware Components

Von Neumann Machines
With rare exceptions, all computers today have the same general IAS structure and function — they are "von Neumann machines."

Opcode = 8 bits → max 256 instructions (IAS used 21). Address = 12 bits → 212 = 4,096 addressable locations. Only one MAR, one MBR — "indirection" simplifies electronics.

IAS Instruction Cycle

Fetch Cycle — Load Next Opcode into IR
  1. Check IBR: is next instruction already buffered? If yes, skip to step 4
  2. Load instruction pair from memory: MAR ← PC; MBR ← M(MAR); PC ← PC + 1
  3. For left instruction: IR ← MBR(0:7); MAR ← MBR(8:19); IBR ← MBR(20:39)
  4. For right instruction (from IBR): IR ← IBR(0:7); MAR ← IBR(8:19)
Execute Cycle — Perform Instruction
  1. Decode opcode in IR using control circuitry
  2. Send appropriate control signals
  3. Move data or perform ALU operation per decoded instruction
Example Executions
  • LOAD M(X): MBR ← M(MAR); AC ← MBR
  • ADD M(X): MBR ← M(MAR); AC ← AC + MBR
  • JUMP+ M(X,0:19): if AC > 0 then PC ← MAR
Indirection Reason
Only ONE MAR (for specifying memory addresses) and ONE MBR (source/destination). This simplifies the electronics by limiting the number of data paths required.

1st Generation: Vacuum Tubes (1946–1957)

Key Technologies
  • Vacuum tubes for digital logic and memory
  • Require wires, metal plates, glass capsule, and vacuum — physically large and fragile
  • High power consumption and heat generation
IAS Computer (Landmark 1st Gen)
  • Designed 1946, completed 1952, Princeton Institute for Advanced Studies
  • Stored-program concept: von Neumann + Turing
  • Binary representation of data and instructions
  • 4,096 words × 40-bit memory
Transistor Discovery
  • Transistor invented at Bell Labs, 1947
  • Solid-state device made from silicon — no vacuum required
  • Smaller, cheaper, generates less heat than vacuum tube
  • By late 1950s: fully transistorized commercial computers available
Fundamental Computer Elements
Only two types needed: Gates (implement Boolean functions, process data) and Memory Cells (store 1 bit, two stable states). All computers are built from these.

2nd Generation: Transistors (1957–1964)

Advances Over Vacuum Tubes
  • Smaller, cheaper, less heat, solid-state (silicon)
  • More complex ALU and control units
  • Introduction of high-level programming languages
  • System software (OS precursors): load programs, move data to peripherals, common libraries
IBM 7094 (Important 2nd Gen System)

Data Channels
CPU sends control signal to data channel → channel executes I/O sequence independently → signals CPU when done. Dramatically reduces CPU burden for I/O operations.

3rd Generation: Integrated Circuits (1965–1971)

1958: Invention of integrated circuit — single-chip fabrication of an entire circuit in silicon
IC Technology
  • Microelectronics: fabricate entire circuit in tiny piece of silicon
  • Transistors, resistors, conductors all from semiconductor (silicon)
  • Many transistors produced simultaneously on single silicon wafer
  • Metallization connects transistors into circuits
  • Wafer → matrix of small areas (few mm²) → broken into individual chips
  • Each chip packaged with input/output attachment points
IBM System/360 (1964)
  • Industry's first planned family of computers
  • Incompatible with older IBM machines — controversial but necessary
  • IBM captured >70% market share; architecture survived to IBM mainframe line today
  • Family: same ISA across models; only speed/cost differs
  • Customer could buy cheap Model 30 → later upgrade without losing software
DEC PDP-8 (1964)
  • Used a single shared bus (Omnibus) connecting all components
  • Revolutionary flexible, expandable architecture
  • Forerunner of modern bus-based computer architectures

Moore's Law

Gordon Moore (Intel co-founder), 1965: transistors per chip doubles every year → revised to every 18 months in 1970s
Transistors(t) ≈ Transistors(0) × 2^(t / 1.5 years) Example: 1978 8086 = 29,000 transistors 2013 Core i7 = 1,860,000,000 transistors Ratio ≈ 64,138× over ~35 years
5 Key Consequences of Moore's Law
  1. Lower cost: cost per chip unchanged while density grows → cost of logic/memory falls dramatically
  2. Higher speed: closer components → shorter electrical paths → faster operation
  3. Smaller computers: more convenient for varied environments
  4. Less power: reduced energy requirements per computation
  5. More reliable: fewer inter-chip solder connections; metallization more reliable than solder joints
Intel Tick-Tock Model
Intel's strategy to maintain Moore's Law pace: alternating between new silicon technology (Tick) and new processor microarchitecture (Tock) on alternating years.

Later Generations: LSI → VLSI → ULSI

4th Generation (1972–1977): LSI
  • Large Scale Integration: thousands of transistors per chip
  • 10 million operations per second
  • Intel introduced first microprocessors (4004 in 1971, 8008 in 1972)
  • Personal computing begins to emerge
5th Generation (1978–1991): VLSI
  • Very Large Scale Integration: millions of transistors per chip
  • 100 million operations per second
  • Intel 8086 (1978) — foundation of x86 architecture
  • IBM PC (1981) using Intel 8088
  • Complex instruction sets, hardware caches, FPUs
6th Generation (1991–): ULSI
  • Ultra Large Scale Integration: billions of transistors per chip
  • >1 billion operations per second
  • Multi-core processors, GPU computing, deep pipelining
  • Today: 2.75B transistors on a single chip (IBM zEC12)

Intel x86 — CISC Architecture Overview

CISC = Complex Instruction Set Computer — sophisticated design principles once found only in mainframes/supercomputers
x86 Architecture Characteristics
  • Intel #1 microprocessor maker for non-embedded systems for decades
  • Full backward compatibility: any program from older x86 runs on newer versions
  • All changes = additions to instruction set — no removals ever
  • ~1 new instruction added per month → now thousands of instructions [ANTH08]

Embedded Systems — §1.5

Embedded system: use of electronics and software within a product, as opposed to a general-purpose computer
Scale Comparison
Millions of general-purpose computers sold per year. Billions of embedded computer systems produced per year. Today, most electric-powered devices contain an embedded computing system.
Real-Time Constraints
  • Tightly coupled to environment → real-time constraints
  • Reactive system: continual interaction with environment; executes at pace determined by environment
  • Constraints include: required speeds of motion, precision of measurement, time durations
  • Multiple simultaneous activities → more complex real-time requirements
Differences from Desktop Computers
  • Sensors and actuators for environmental interaction
  • Human interface: simple (flashing light) → complex (robotic vision) or none at all
  • Diagnostic port: diagnoses the controlled system, not just the computer
  • Special hardware: FPGA, ASIC, or non-digital hardware for performance/reliability
  • Fixed-function software specific to the application
  • Optimization targets: energy, code size, execution time, weight/dimensions, cost
Example Applications

Cell phones · Digital cameras · Microwave ovens · Thermostats · Printers · Automotive (ABS, cruise control, fuel injection, transmission) · Industrial automation · Home security · Smart TVs

Internet of Things (IoT)

IoT: expanding interconnection of smart devices, from appliances to tiny sensors — billions of embedded devices
IoT Characteristics
  • Short-range mobile transceivers embedded in wide array of gadgets
  • New forms of communication: people↔things, things↔things
  • Internet supports interconnection of billions of industrial and personal objects via cloud
  • Objects deliver sensor information, act on environment, modify themselves
  • Enables management of larger systems (factories, cities)
4 Generations of Internet Deployment
  1. Information Technology (IT): PCs, servers, routers, firewalls — enterprise IT, primarily wired
  2. Operational Technology (OT): medical machinery, SCADA, process control — enterprise OT, primarily wired
  3. Personal Technology: smartphones, tablets, eBooks — consumer devices, wireless only
  4. Sensor/Actuator Technology (IoT): single-purpose devices, consumer/IT/OT, wireless only — billions of devices
IoT Data Characteristics
  • Primarily low-bandwidth, low-repetition data-capture appliances
  • High-bandwidth exceptions: video security cameras, video VoIP phones
  • Driven primarily by deeply embedded devices

Deeply Embedded Systems

Deeply embedded system: processor whose behavior is difficult to observe by programmer and user; ROM-programmed; no user interaction
Defining Characteristics
  • Uses microcontroller (not microprocessor)
  • Not reprogrammable: program logic burned into ROM (read-only memory)
  • No user interaction: completely autonomous operation
  • Dedicated single-purpose device
  • Processor behavior difficult to observe
Function Pattern
  1. Detect something in the environment (via sensors)
  2. Perform a basic level of processing
  3. Do something with the results (via actuators)
Resource Constraints
  • Memory: extremely limited (KB range)
  • Processor size: minimal gate count
  • Time: strict real-time deadlines
  • Power: often battery-powered; ultra-low consumption
Deployment Patterns
  • Often have wireless capability
  • Deployed in networked configurations (sensor networks)
  • Factory sensor networks, agricultural field arrays
  • IoT depends heavily on deeply embedded systems

ARM Architecture — Overview & History

ARM = Advanced RISC Machines (originally Acorn RISC Machine) — most widely used processor architecture in the world
ARM Holdings Business Model
  • Designs microprocessor and multicore architectures
  • Does NOT manufacture chips — licenses designs to manufacturers
  • Processor license: customer uses ARM-supplied design in their chips
  • Architecture license: customer designs own processor compliant with ARM architecture
Historical Timeline
YearEvent
Early 1980sAcorn Computers awarded BBC contract (Computer Literacy Project)
1985ARM1 — first commercial RISC processor; used as BBC coprocessor
Later 1980sVLSI Technology licensed to fabricate/market chips; early embedded success
~1990ARM Ltd. formed: Acorn + VLSI + Apple Computer as founding partners
Late 1990sDropped "Advanced RISC Machines" name; simply "ARM architecture"
Key Characteristics
  • High-speed, small die size, low power requirements
  • Apple iPod and iPhone processors; virtually all Android smartphones
  • Dominant in embedded systems globally [VANC14]

ARM ISA & Cortex Product Family

ARM Instruction Set Architecture
  • All instructions 32 bits long with regular, uniform format
  • Designed for efficient implementation across wide product range
  • Thumb ISA: 16-bit re-encoded subset of ARM 32-bit ISA; better code density; for 16-bit or narrower memory bus
  • Thumb-2: current version; mixed 16/32-bit instructions
  • Harvard architecture (Cortex-M3): separate instruction and data buses → parallel access

MMU vs MPU
MMU (Memory Management Unit): translates virtual→physical addresses; required for full OS (Linux, Android). MPU (Memory Protection Unit): prohibits programs from accessing each other's memory; simpler; used in Cortex-R/M.

Cloud Computing — §1.7

NIST SP-800-145: "A model for enabling ubiquitous, convenient, on-demand network access to a shared pool of configurable computing resources that can be rapidly provisioned and released with minimal management effort or service provider interaction."
Historical Context
  • General concepts from 1950s; services first available early 2000s
  • Initially targeted large enterprises; spread to SMBs, then consumers
  • Apple iCloud (2012): 20 million users in first week
  • Evernote (2008): approached 100 million users in <6 years
Core Benefits
  • Economies of scale: shared infrastructure reduces per-user cost
  • Professional network management: high-performance, high-reliability
  • Professional security management: dedicated security expertise
  • Pay only for storage capacity and services needed
  • No setup, maintenance, backup, or hardware acquisition burden
Cloud Subcategories
  • Cloud Storage: database storage and applications hosted remotely; scales with needs
  • Cloud Networking: networks enabling cloud access; high-performance links; firewalls at critical points; may bypass public Internet via private facilities
Security Concern
Security failures among cloud providers have occurred (e.g., Evernote 2013 password reset after intrusion). Customer is not always fully protected despite provider's security management.
