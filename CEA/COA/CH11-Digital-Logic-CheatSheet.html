Boolean Algebra — Postulates & Theorems

Boolean algebra is a two-valued (0/1) algebraic system used to describe, simplify, and implement digital logic circuits. Variables take only values 0 or 1; three basic operations: AND (·), OR (+), NOT (').

Identity & Null Laws
A + 0 = A    A · 1 = A    (Identity)
A + 1 = 1    A · 0 = 0    (Null / Dominance)
Idempotent & Inverse
A + A = A    A · A = A    (Idempotent)
A + A' = 1    A · A' = 0    (Inverse / Complement)
A'' = A    (Double Negation / Involution)
Commutative & Associative
A + B = B + A    A · B = B · A
(A + B) + C = A + (B + C)
(A · B) · C = A · (B · C)
Distributive & Absorption
A · (B + C) = A·B + A·C    (AND over OR)
A + (B · C) = (A+B) · (A+C)    (OR over AND)
A + A·B = A    (Absorption)
A · (A + B) = A    (Absorption)
DeMorgan's Theorems
(A · B)' = A' + B'    NAND → NOR of complements
(A + B)' = A' · B'    NOR → NAND of complements
Canonical Forms
SOP (Sum of Products): F = A'B + AB' + AB
POS (Product of Sums): F = (A+B)·(A'+B)·(A+B')
Simplification
Use Boolean postulates or Karnaugh maps (K-maps) to minimize the number of gates. SOP → OR of minterms; POS → AND of maxterms.

Fan-In / Fan-Out
Fan-in = max inputs a gate accepts. Fan-out = max gates driven by one output without signal degradation.

Key Observations
  • AND: output 1 only when A=1 AND B=1 (1 out of 4 cases)
  • OR: output 0 only when A=0 AND B=0 (3 out of 4 cases → 1)
  • NAND: complement of AND; output 0 only for A=B=1
  • NOR: complement of OR; output 1 only for A=B=0
  • XOR: output 1 when inputs differ (A≠B)
  • XNOR: output 1 when inputs are equal (A=B)
Minterms vs Maxterms
Minterm m_i: AND term that = 1 for row i only
Maxterm M_i: OR term that = 0 for row i only
Canonical SOP
F = Σm(minterms where F=1). For XNOR: F = Σm(0,3) = A'B' + AB.
Canonical POS
F = ΠM(maxterms where F=0). For XOR: F = ΠM(0,3) = (A+B)·(A'+B').

Universal Gates & Functional Completeness

A gate is universal if any Boolean function can be implemented using only that gate type.

NAND as Universal Gate
NOT from NAND: A'
A ─┬─ NAND ─ A'
A ─┘ (tie both inputs)
AND from NAND: A·B
NAND(A,B) → NAND with itself
= ((A·B)')' = A·B ✓
OR from NAND (DeMorgan's)
NAND(A',B') = (A'·B')' = A+B
Invert each input, then NAND
NOR as Universal Gate
NOT, AND, OR from NOR
NOT: NOR(A,A) = (A+A)' = A'
OR: NOR(NOR(A,B),NOR(A,B)) = A+B
AND: NOR(A',B') = (A'+B')' = A·B
XOR Properties
  • A⊕0 = A   (identity)
  • A⊕1 = A'   (complement)
  • A⊕A = 0   (self-inverse)
  • A⊕A' = 1   (always true)
  • Associative: (A⊕B)⊕C = A⊕(B⊕C)
  • Used in: parity, adders, encryption

Combinational Circuits — Overview

A combinational circuit has outputs that depend only on current inputs — no memory, no feedback, no clock needed.

Key Types
  • Adders — half adder, full adder, RCA, CLA
  • Multiplexers — data selector (MUX)
  • Demultiplexers — data distributor (DEMUX)
  • Decoders — n → 2ⁿ, one-hot output
  • Encoders — 2ⁿ → n, priority encoder
  • Comparators — A=B, A>B, A<B outputs
  • ALU — combines add, sub, AND, OR, shift
  • Parity generators/checkers — XOR trees
Design Process
  1. Define inputs, outputs, and behavior in words
  2. Construct truth table for all input combinations
  3. Write Boolean expression (SOP or POS)
  4. Simplify using Boolean algebra or K-map
  5. Implement with gates; verify timing constraints
Propagation Delay
Real gates introduce delay (t_pd). In chains, delays add; critical path = longest delay path, limits clock speed.

Adds two 1-bit inputs A and B. Produces Sum and Carry out. No carry-in — only usable for the LSB position.

Propagation Delay
RCA delay = n × t_FA. For 32-bit adder: 32 × gate delays. Slow for wide buses.
Carry Lookahead Adder (CLA)

Computes all carries in parallel using generate and propagate signals:

G_i = A_i · B_i    (generate carry)
P_i = A_i ⊕ B_i    (propagate carry)
C_{i+1} = G_i + P_i·C_i
CLA Advantage
All carry bits computed simultaneously in O(1) logic levels (2 gate delays). Much faster than RCA for wide adders.

Programmable Logic: ROM, PLA, PAL, FPGA

Programmable devices implement arbitrary combinational (and sequential) logic without wiring discrete gates.

Comparison Table
DeviceAND ArrayOR ArrayFlexibility
ROMFixedProgrammableLow
PLAProgrammableProgrammableHighest
PALProgrammableFixedMedium
FPGALUT-based CLBsVery high
ROM (Read-Only Memory)
  • n address inputs → 2ⁿ word locations, each m bits wide
  • Implements any truth table: address = input, data = output
  • AND array is complete decoder (fixed); OR array is user-programmable
  • Use: lookup tables, microprogram stores, boot code
PLA (Programmable Logic Array)
  • Both AND and OR planes are programmable
  • Implements sum-of-products for multiple outputs
  • More area-efficient than ROM (only needed minterms)
PAL (Programmable Array Logic)
  • Programmable AND plane, fixed OR plane
  • Simpler to program than PLA; most widely used in practice
  • Some PALs include registered (flip-flop) outputs for sequential logic
FPGA (Field-Programmable Gate Array)
  • Array of configurable logic blocks (CLBs) + interconnect
  • CLB contains LUT (look-up table), flip-flop, carry chain
  • Reprogrammable; used for prototyping and production

Sequential Circuits — Overview

A sequential circuit has outputs that depend on both current inputs and the circuit's past history (internal state). State is stored in flip-flops.

Key Characteristics
  • Requires a clock signal for synchronization
  • State register: flip-flops hold current state Q
  • Next-state logic: combinational circuit computes Q_next from Q and inputs
  • Output logic: combinational circuit computes outputs
Synchronous vs Asynchronous
TypeState changesClock
SynchronousOnly on clock edgeRequired
AsynchronousAny time (input change)Not used
Mealy vs Moore Machine
ModelOutput depends onOutput changes
MealyState + current inputsImmediately with inputs
MooreState onlyOnly on clock edge
State Machine Design
Define states → state transition diagram → state table → flip-flop excitation equations → implement.
Timing Parameters
  • Setup time (t_su): input must be stable BEFORE clock edge
  • Hold time (t_h): input must be stable AFTER clock edge
  • Propagation delay (t_pd): clock edge to output change
  • Clock period: must be > t_pd + t_su + routing delay

Edge-Triggered Summary
Level vs Edge
Level-triggered (latch): transparent while CLK=1. Edge-triggered (FF): captures only on ↑ or ↓ clock edge. Edge-triggered is preferred in synchronous design to prevent races.

Registers

A register is a group of flip-flops that stores an n-bit word. All FFs share a common clock and (optionally) a load/enable control.

Parallel Load Register
  • All n bits loaded simultaneously on one clock edge
  • Load signal (LD): LD=1 → capture data; LD=0 → hold
  • Uses D flip-flops: D_i = LD·Data_i + LD'·Q_i
  • Most common: CPU general-purpose registers (R0–R15)
Shift Registers

Each FF's output feeds the next FF's input. Data moves one position per clock cycle.

Universal Shift Register
  • Can: shift left, shift right, parallel load, hold
  • Controlled by 2 mode select bits (S1, S0)
  • S1S0=00: hold; 01: shift right; 10: shift left; 11: parallel load
Applications
  • Serial communication (UART): PISO + SIPO pair
  • Multiplication by 2 (left shift), division by 2 (right shift)
  • CRC computation, pseudo-random number generation (LFSR)

Counters

A counter is a sequential circuit that cycles through a defined sequence of states on each clock pulse.

Ripple Counter (Asynchronous)
  • MSB FF clocked by system clock; each subsequent FF clocked by Q of previous
  • Carries ripple through chain → increasing delay with bit width
  • Simple, low power; glitches present during propagation
3-bit ripple counter sequence
000 → 001 → 010 → 011
100 → 101 → 110 → 111 → 000
FF0 toggles every cycle; FF1 toggles every 2; FF2 every 4
Synchronous Counter
  • All FFs share the same clock; no ripple delay
  • Faster, glitch-free; uses combinational logic for each FF's J/K or T input
  • T₀=1 always; T₁=Q₀; T₂=Q₁·Q₀; T₃=Q₂·Q₁·Q₀ (binary)
Modulus Counters
CounterSequenceReset condition
Mod-N0 to N-1Count = N → clear
BCD (Mod-10)0000–1001Count = 1010 → clear
Mod-6000–101Count = 110 → clear
Up/Down Counter
  • Direction control input: UP=1 → count up; UP=0 → count down
  • T input conditioned on direction and lower bits
Johnson & Ring Counters
  • Ring counter: single 1 rotates through n bits; Mod-n with n FFs
  • Johnson (twisted ring): Q' of last FF feeds first FF; 2n states from n FFs; easy decode
Counter Applications
Clock dividers, event counters, address generators, PWM, timing circuits, program counters in CPUs.

Memory Elements — SRAM, DRAM, ROM Types

Memory is organized as an array of cells. An address selects a row (word line) and column selector reads/writes data bits.

SRAM Cell Operation
  • 6 transistors form a cross-coupled inverter pair (bistable)
  • Word line selects row; bit lines (BL, BL') read or write
  • Static: holds data as long as power is on, no refresh
DRAM Cell Operation
  • 1 transistor + 1 capacitor per cell (1T1C)
  • Capacitor stores charge (1) or no charge (0)
  • Leaks charge → must refresh periodically (read+rewrite)
  • Row access (RAS) then column access (CAS) with separate strobes

Memory Cell Array Organization
Address bits → Row decoder (2^r rows) + Column selector

Memory Hierarchy Fit
SRAM → L1/L2/L3 cache (fast, expensive). DRAM → main RAM (slow, cheap, dense). Flash → SSDs, firmware storage (non-volatile).
