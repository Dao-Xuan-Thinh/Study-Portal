Arithmetic Logic Unit (ALU)

The part of the CPU that actually performs arithmetic and logical operations on data. All other components (control unit, registers, memory, I/O) exist to bring data to the ALU and retrieve results.

ALU Inputs & Outputs
  • Operands — stored in registers, fed into ALU
  • Results — written back to registers
  • Flags — overflow flag (OF), zero flag, etc. stored in 1-bit registers
  • Control signals — specify operation (add, sub, AND, OR…)
Add/Sub Hardware (Fig. 11.6)

Uses a central binary adder. For subtraction, the subtrahend (B) is passed through a twos complementer before reaching the adder. A 1-bit overflow flag stores the result of overflow detection.

Key Properties
  • Sign-Magnitude: MSB = sign bit; two representations of 0; rarely used in ALUs
  • Twos Complement: One representation of 0; range −2n−1 to 2n−1−1; universally used
  • Biased (Excess-N): True value = stored value − bias; used in float exponents; unsigned comparison works
  • Fixed-Point: Radix point assumed at fixed position (right of LSB for integers)

Twos Complement — Core Rules

Range
–2n–1 ≤ x ≤ 2n–1 – 1   (e.g. 8-bit: –128 to +127)
Value Box Interpretation (n-bit)

Positions left-to-right: −2n−1, 2n−2, …, 21, 20. MSB contributes a negative weight.

8-bit 10000011 → −128 + 2 + 1 = −125
Sign Extension (Range Extension)
  • To extend bit length, copy the sign bit into all new left positions
  • Positive: fill with 0s  |  Negative: fill with 1s
  • Example: 4-bit 1011 (−5) → 8-bit 11111011 (−5) ✓
Characteristics (Table 11.1)
  • Unique representation of zero
  • Addition same as for unsigned integers
  • Subtraction: negate subtrahend, then add
  • Overflow: result has opposite sign from both operands

Negation (Twos Complement)

Two-Step Procedure
  1. Complement all bits (bitwise NOT)
  2. Add 1 to the result (treat as unsigned integer)
Example: Negate +18
+18 = 00010010
Bitwise complement → 11101101
Add 1            + 1
Result           11101110 = −18 ✓
⚠ Special Case 1: Zero
00000000 → 11111111 + 1 = 100000000 → carry discarded → 00000000 = 0 ✓
⚠ Special Case 2: Most Negative (−128)
10000000 → 01111111 + 1 = 10000000 → −(−128) = −128 ✗ (no positive equivalent; monitor MSB change!)

Unsigned Binary Multiplication

Paper-and-Pencil Method
  • Generate one partial product per multiplier bit
  • Multiplier bit = 0 → partial product = 0
  • Multiplier bit = 1 → partial product = multiplicand (shifted left)
  • n × n bits → up to 2n-bit product
Example: 1011 × 1101 = 143
        1011 (11)
      × 1101 (13)
        1011 ← ×2⁰
        0000 ← ×2¹
       1011  ← ×2²
      1011   ← ×2³
    10001111 = 143 ✓
Hardware (Fig. 11.8) — Registers: M, A, Q, C
  • M = multiplicand  |  Q = multiplier  |  A = accumulator (init 0)  |  C = carry bit
  • Each cycle: if Q₀=1, A←A+M; then shift C,A,Q right 1
  • Repeat n times. Product in A:Q

Booth's Algorithm (Twos Complement Multiply)

Why Booth's?

Straightforward unsigned multiplication fails for negative (twos complement) multiplicands. Booth's handles all sign combinations and often speeds up multiplication by skipping blocks of 0s or 1s.

Binary Division

Unsigned Division — Restoring Algorithm

Similar to long division. Registers: A (init 0, becomes remainder), Q (dividend → quotient), M (divisor).

  1. Shift A, Q left 1 bit
  2. Subtract M from A: A ← A − M
  3. If A ≥ 0: set Q₀ = 1  |  If A < 0: set Q₀ = 0 and restore A (A ← A + M)
  4. Decrement count; repeat n times
Result: Quotient in Q, Remainder in A
Example: 10010 ÷ 1011 (Fig 11.15)
Dividend = 10010 (18), Divisor = 1011 (11)
Quotient = 00001101 (no wait — Q=0001), Rem=100
18 ÷ 11 = 1 remainder 7 → Quotient=00001, Rem=0111
Twos Complement Division

Convert both operands to positive magnitude, perform unsigned division, then apply signs:

sign(Remainder) = sign(Dividend)
sign(Quotient) = sign(Dividend) × sign(Divisor)

32-bit Float Number Ranges
  • Positive normal: 2−126 to (2 − 2−23) × 2128
  • Negative normal: −(2 − 2−23)×2128 to −2−126
  • Positive underflow: 0 < x < 2−126 (→ result ≈ 0)
  • Overflow: |x| > (2 − 2−23) × 2128
FP Arithmetic Errors
  • Exponent overflow → ±∞
  • Exponent underflow → 0 or subnormal
  • Significand underflow → rounding needed (digits lost right)
  • Significand overflow → carry from MSB → re-align (shift right, increment exponent)

Floating-Point Arithmetic Operations

General Formulae (base B)
X + Y = (XS·BXE−YE + YS) · BYE
X − Y = (XS·BXE−YE − YS) · BYE
X × Y = (XS·YS) · BXE+YE
X ÷ Y = (XS÷YS) · BXE−YE
Add/Subtract — 4 Phases (Fig 11.22)
  1. Zero check — if either operand = 0, return the other
  2. Align significands — shift smaller number right until exponents equal (shift the smaller, losing less precision)
  3. Add/Sub significands — consider signs; if sum = 0 → return 0; handle significand overflow (shift right, increment exponent)
  4. Normalize & round — shift left until leading bit = 1; decrement exponent; check underflow; round result
Multiply (Fig 11.23)

Add exponents (subtract bias once); multiply significands; normalize & round.

Divide (Fig 11.24)

Subtract exponents (add bias back); divide significands; normalize & round.

Precision: Guard Bits & Rounding

Guard Bits

Extra bits appended to the right of significand in ALU registers during computation. Prevent precision loss during alignment (right-shifting smaller operand). Without guard bits, a 1-bit shift can lose the LSB, causing up to a factor-of-2 error (factor-of-16 for hex).

IEEE 754 Rounding Modes
Round to Nearest (default): round to closest representable value; on tie → round to even (last bit = 0)
Round toward +∞: always round up (ceiling)
Round toward −∞: always round down (floor)
Round toward 0: truncation — extra bits dropped; simplest but introduces bias
Interval Arithmetic

Uses "round toward +∞" and "round toward −∞" simultaneously to compute a bounding interval [lower, upper] for every result. Width of interval = accuracy estimate.

Truncation (Round toward 0)
  • Simplest but introduces systematic bias toward zero
  • Affects every operation with nonzero extra bits

Subnormal Numbers & Gradual Underflow

Problem Without Subnormals

In 32-bit IEEE 754, the smallest normal positive number is 2−126. Without subnormals, there's a large gap between 0 and 2−126 — a number slightly below 2−126 would abruptly underflow to 0.

Subnormal Format
Exponent field = 0 AND fraction ≠ 0 → subnormal

True exponent is treated as −126 (binary32), but the hidden leading bit is 0 (not 1):

Value = ±2−126 × 0.fraction
Effect — Gradual Underflow (Fig 11.26)

Adds 223−1 additional numbers uniformly between 0 and 2−126. The spacing between consecutive representable numbers near zero matches the spacing in the first normal interval [2−126, 2−125].

Smallest Positive Values (Binary32)
  • Smallest normal: 2−126 ≈ 1.175 × 10−38
  • Smallest subnormal: 2−149 ≈ 1.401 × 10−45
💡 Benefit
Gradual underflow reduces the impact of exponent underflow to a level comparable with ordinary rounding error.
