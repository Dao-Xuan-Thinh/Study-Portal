§6.1 — Magnetic Disk

Disk Structure & Anatomy

Physical Components
  • Platter — Circular non-magnetic substrate (aluminum alloy or glass) coated with magnetizable material
  • Track — Concentric ring on platter surface; thousands of tracks per surface; same width as R/W head
  • Sector — Arc segment of a track; typical size = 512 bytes (or 4096 bytes in modern drives)
  • Cylinder — Set of all tracks at the same radial position across all platters
  • Spindle — Central shaft that rotates all platters together
  • Boom / Arm — Mechanical arm that positions the R/W head radially
  • Intertrack gap — Space between adjacent tracks (prevents cross-track interference)
  • Intersector gap — Space between adjacent sectors on a track
Glass Substrate Advantages
  • More uniform magnetic film → fewer read/write errors
  • Fewer surface defects
  • Supports lower fly heights (greater density)
  • Better stiffness, greater shock resistance

Read / Write Mechanisms

Write (Inductive)

Electric pulses → conducting coil → magnetic field across gap → magnetizes surface. Reversing current direction reverses magnetization polarity (encodes 0/1).

Read (Traditional)

Moving magnetic field induces current in coil. Same head structure used for both read and write (floppy/older rigid disks).

Read (Modern — MR Sensor)

Separate Magnetoresistive (MR) read head. Electrical resistance changes based on magnetic direction of passing medium → detected as voltage signals. Allows higher frequency → greater density & speed.

Key Point
During read/write, the head is stationary — the platter rotates beneath it.
Data Encoding
  • Change between pit/land states represents bit boundaries
  • No change between sensor intervals → 0
  • Beginning or end of pit → 1

Physical Characteristics

DimensionOptions

Winchester Disk

Sealed drive assembly (near-contaminant-free). Aerodynamic foil rests on surface when idle; air pressure from spinning disk lifts head (~0.1 µm). Allows narrow heads → greater data density.

Cylinder Note
All heads move together → they always sit over the same cylinder. Accessing data on the same cylinder requires NO additional seek.

Data Layout: CAV vs MZR

CAV — Constant Angular Velocity

Disk rotates at fixed speed. All tracks hold the same number of bits. Outer track bits are spaced farther apart (lower linear density). Easy direct addressing by track + sector. Wastes outer track capacity.

MZR — Multiple Zone Recording

Surface divided into ~16 concentric zones. Within each zone, bit count per track is constant. Outer zones have more sectors per track than inner zones → approximately uniform linear bit density → greater overall capacity.

Trade-off
MZR: More complex circuitry (timing changes when head crosses zone boundary) but significantly higher storage capacity.
Optical Disks (CD/DVD)

Use a single spiral track with CLV (Constant Linear Velocity): disk rotates slower for outer areas, faster for inner areas. All sectors same physical length → maximum density but harder random access.

Disk Performance Parameters

Three Components of Access Time
T_access = T_seek + T_rotation + T_transfer
Seek Time (T_seek)

Time to move arm to target track. Two components: startup time + traversal time (non-linear; includes settling time). Typical average: < 10 ms on modern drives.

Rotational Latency (T_rotation)

Time waiting for target sector to rotate under head. Average = ½ rotation period.

T_rotation_avg = 1 / (2r) where r = rev/sec

Example: 15,000 rpm → r = 250 rev/s → avg latency = 2 ms

Transfer Time (T_transfer)
T_transfer = b / (r × N)

b = bytes to transfer; N = bytes per track; r = rev/sec

Full Formula (Eq. 6.1)
T_total = T_s + 1/(2r) + b/(rN)
Additional Delays (in practice)
  • Wait for device — queued behind other processes
  • Wait for channel — shared I/O channel busy
  • RPS miss — rotational positional sensing miss → extra full revolution penalty

§6.2 — RAID (Redundant Array of Independent Disks)

RAID Overview & Motivation

Why RAID?

CPU/DRAM performance improves much faster than disk I/O. RAID uses multiple parallel disks to close this gap — improving both performance and reliability.

3 Common Characteristics of All RAID Levels
  1. Multiple physical disks appear as a single logical drive to the OS
  2. Data distributed across drives via striping (round-robin allocation of strips)
  3. Redundant capacity stores parity information for data recoverability
    (Exception: RAID 0 has no redundancy; RAID 1 uses mirroring not parity)
Key Terminology
  • Strip — unit of data on a single disk (can be sector, block, or user-defined)
  • Stripe — one strip per disk in the array (n strips forming a logical row)
  • Parallel access — all disks participate in every I/O (RAID 2, 3)
  • Independent access — disks can serve separate I/O requests (RAID 4, 5, 6)
Origin
Coined by Patterson, Gibson, Katz at UC Berkeley (1988). Originally "Inexpensive Disks" → now "Independent Disks."

RAID 0 & RAID 1

RAID 0 Striping — No Redundancy
  • Data striped round-robin across N disks. No parity/mirroring.
  • Multiple concurrent I/O requests → served from different disks in parallel
  • Single large I/O → split across all N disks → N× effective transfer rate
  • Disks required: N (no overhead)
  • Storage efficiency: 100%
Critical Risk
One disk failure = ALL data lost. No fault tolerance whatsoever.
RAID 1 Mirroring
  • Every strip written to TWO disks simultaneously (mirror copy)
  • Read: served from either disk → choose lowest seek + latency → up to 2× read rate
  • Write: both disks updated in parallel → speed of the slower write (no extra parity overhead)
  • Disk failure: data fully available from mirror instantly
  • Disks required: 2N
  • Storage efficiency: 50% (100% overhead)
Best Use Cases
RAID 0: video production, image editing (high bandwidth, tolerance for risk)
RAID 1: OS drives, critical databases, financial systems

RAID 2 & RAID 3 (Parallel Access)

RAID 2 Hamming Code ECC
  • Very small strips (single byte or word). All disks synchronized (spindles aligned).
  • Error-Correcting Code (Hamming) computed across corresponding bits of each data disk
  • ECC bits stored on separate m parity disks (m ∝ log₂N)
  • Can correct single-bit errors, detect double-bit errors
  • Disks required: N + m
Practical Note
High ratio of ECC to data disks for small arrays. Modern drives have built-in ECC → RAID 2 is not commercially used.
RAID 3 Bit-Interleaved Parity
  • Similar to RAID 2 but only 1 dedicated parity disk regardless of array size
  • Very small strips (bit/byte level). All disks participate in every I/O.
  • Parity: P(i) = X3(i) ⊕ X2(i) ⊕ X1(i) ⊕ X0(i)
  • Failed drive data: Xi(j) = P(j) ⊕ (all other disks' j-th bit)
  • Excellent for large sequential transfers; poor for many small random I/Os
  • Disks required: N + 1 · Efficiency: N/(N+1)
Reduced Mode
In case of failure, missing data are regenerated on-the-fly via XOR. Full recovery only after physical disk replacement.

RAID 4 — Block-Level Parity

RAID 4 Independent Access + Dedicated Parity
  • Large strips (block-level). Independent access: each disk can serve a separate I/O request.
  • Bit-by-bit parity computed across corresponding strips of each data disk
  • Parity stored on one dedicated parity disk
Write Penalty (Small Writes)
New_P = Old_P ⊕ Old_Data ⊕ New_Data

Each small write requires: 2 reads (old data + old parity) + 2 writes (new data + new parity) → 4 disk operations per logical write

Bottleneck
Every write involves the parity disk → parity disk becomes a bottleneck under heavy write loads. Not commercially implemented.
Specs
  • Disks required: N + 1
  • Storage efficiency: N/(N+1)
  • Read performance: Similar to RAID 0
  • Write performance: Much lower than single disk

RAID 5 — Distributed Parity ⭐

RAID 5 Block-Interleaved Distributed Parity
  • Like RAID 4 but parity strips distributed round-robin across all disks
  • First N stripes: parity on disk 0, 1, …, N-1; pattern repeats
  • Eliminates the RAID 4 parity disk bottleneck
  • Independent access: multiple I/O requests served in parallel
Specs
  • Disks required: N + 1 (min. 3)
  • Storage efficiency: N/(N+1)
  • Tolerates: 1 disk failure
Most Popular RAID Level
Best balance of performance, storage efficiency, and fault tolerance. Used in file servers, database servers, web servers, and intranet servers. Most versatile RAID level.
RAID 10 (1+0)
Combination: stripe across mirrored pairs. Provides RAID 0 performance with RAID 1 redundancy. Requires 2N disks. Popular in high-demand environments.

RAID 6 — Dual Distributed Parity

RAID 6 Block-Interleaved Dual Distributed Parity
  • Two independent parity calculations (P and Q) stored on separate disks
  • P = standard XOR parity (same as RAID 5)
  • Q = independent second check algorithm (e.g., Reed-Solomon)
  • Can recover from any 2 simultaneous disk failures
Specs
  • Disks required: N + 2 (min. 4)
  • Storage efficiency: N/(N+2)
  • Read performance: Similar to RAID 0 / RAID 5
  • Write performance: Each write affects 2 parity blocks → >30% write penalty vs RAID 5
When to Use
Ideal For
Mission-critical applications, large disk arrays (where double-failure probability is non-negligible), archival storage, disaster recovery systems.
Write Overhead
Performance benchmarks show RAID 6 controllers suffer 30%+ write performance drop compared to RAID 5.

SSD Architecture & Organization

SSD System Architecture (Fig 6.8)
OS Software → File System → I/O Driver → [Interface] → SSD Controller
SSD Internal Components
  • Controller — Device-level interfacing; runs firmware; manages all internal ops
  • Addressing Logic — Selection function across flash memory chips; manages logical-to-physical mapping (FTL)
  • Data Buffer / Cache — High-speed RAM for speed-matching; increases throughput
  • Error Correction Logic — ECC for detecting and correcting read errors
  • NAND Flash Chips — Actual non-volatile storage (arrays of floating-gate transistors)
Host Interfaces
  • PCIe / NVMe — Internal high-performance SSDs (ultra-low latency)
  • SATA — Legacy interface, 600 MB/s max, common in older laptops/desktops
  • USB — External SSDs
Flash Memory Hierarchy
  • Page — Smallest read/write unit; typically 4 KB
  • Block — Smallest erasable unit; typically 512 KB (~128 pages)
  • Flash can be read/written at page level but erased only at block level

SSD Practical Issues & Solutions

Issue 1: Write Amplification / Performance Degradation

Writing a single page requires read-modify-write at the block level:

  1. Read entire block (512 KB) into RAM buffer
  2. Update the target page in RAM buffer
  3. Erase the entire block in flash memory
  4. Write entire block from RAM back to flash

As drive fills up → more fragmentation → more blocks affected per write → performance degrades.

Solutions to Degradation
  • Over-provisioning — Reserve extra flash capacity (not exposed to OS) for write operations
  • Garbage Collection — During idle time, consolidate valid pages, erase stale blocks
  • TRIM command — OS informs SSD which blocks are no longer in use → SSD can pre-erase them
Issue 2: Wear-Out (Limited P/E Cycles)

Flash cells degrade after repeated write/erase cycles. Typical limit: ~100,000 P/E cycles.

Solutions to Wear-Out
  • Wear Leveling — Distribute writes evenly across all blocks (FTL algorithm)
  • Write Caching — Front-end RAM cache delays/groups write operations
  • Bad Block Management — Map out worn cells; maintain spare blocks
  • RAID for SSDs — Further reduce probability of data loss
  • Self-monitoring — Drives estimate remaining lifetime; trigger preemptive alerts
§6.4 — Optical Memory

CD Technology: CD-ROM, CD-R, CD-RW

CD Construction & Read Mechanism
  • Polycarbonate resin disk; digitized data as microscopic pits (low reflectivity) and lands (high reflectivity)
  • Coated with reflective aluminum/gold layer + clear acrylic protective coat
  • 780-nm laser reads pit/land transitions; photosensor detects intensity changes
  • Transition (pit↔land) = 1 ; no change between intervals = 0
Spiral Track + CLV

Single spiral track from center outward. Uses CLV (Constant Linear Velocity): disk spins faster for inner tracks, slower for outer tracks. All sectors same physical length → maximum density. Capacity: ~680 MB.

CD-ROM Block Format (Fig 6.10) — 2352 bytes/block
FieldSizeDescription
Sync12 B0x00, 10×0xFF, 0x00 — block delimiter
ID/Header4 BAddress (min, sec, sector) + mode byte
Data2048 BUser data (Mode 1) or 2336 B (Mode 2)
L-ECC288 BLayered error-correction code (Mode 1)
CD-R (Write-Once)

Dye layer activated by high-intensity laser → permanent change in reflectivity. Readable on standard CD-ROM drives. Good for archival storage.

CD-RW (Rewritable)

Uses phase-change material: amorphous state (low reflectivity) ↔ crystalline state (high reflectivity), switched by laser power. Rewritable 500K–1M times. Eventually loses phase-change properties permanently.

DVD & High-Definition Optical Disks

DVD Capacity Improvements vs CD
  1. Smaller pits & tighter tracks — DVD: pit spacing 0.4 µm / track pitch 0.74 µm vs CD: 0.834 µm / 1.6 µm → ~7× capacity → 4.7 GB per layer per side
  2. Dual layers — Second semireflective layer on top → read by adjusting laser focus → ~8.5 GB (single side, dual layer)
  3. Double-sided — Data on both sides → max 17 GB (dual-layer, double-sided)

Optical Memory: Pros, Cons & CLV vs CAV

CD-ROM Advantages
  • Mass replication is cheap — stamp from master disk
  • Removable media — disk itself IS the archive; no drive needed to store data
  • Very high durability for read-only data
  • Standardized format — universal compatibility
CD-ROM Disadvantages
  • Read-only — cannot update existing data
  • Access time much longer than HDD (up to 0.5 seconds)
  • CLV makes random access difficult (must adjust speed + find sector)

§6.5 — Magnetic Tape & LTO

Magnetic Tape Technology

Physical Structure
  • Flexible polyester tape coated with magnetizable material (pure metal particles or vapor-plated films)
  • Widths: 0.38 cm (digital cameras) to 1.27 cm (LTO standard)
  • Housed in cartridges (open reels now obsolete)
  • Data organized as parallel tracks running lengthwise
Recording Methods Evolution
EraMethodDetails
EarlyParallel recording9 tracks: 8 data + 1 parity (one byte at a time); later 18 or 36 tracks
ModernSerial recordingBits laid sequentially along each track (like disk)
Serpentine Recording
  • Record along full tape length → reverse at end → record new track in opposite direction → repeat
  • Multiple adjacent tracks written simultaneously (typically 2–8) for speed
  • Blocks on adjacent tracks interleaved sequentially (Fig 6.13b)
Sequential vs Direct Access
Sequential-Only Access
Tape is a sequential-access device. To read record N, must read all records 1 through N-1 first. If head is past the target, must rewind. Tape moves ONLY during read/write (vs disk which spins continuously).
Role in Memory Hierarchy
Tape = lowest cost, slowest speed member of the memory hierarchy. Ideal for backup, archival, and disaster recovery at massive scale.

LTO (Linear Tape-Open) Generations

LTO Overview

Dominant tape standard developed in late 1990s as open-source alternative to proprietary systems. All versions use 1.27 cm wide tape.

Key Characteristics of External Memory
  • Non-volatile — Data persists without power
  • Large capacity — Orders of magnitude more than RAM
  • Low cost per bit — Magnetic tape = cheapest; tape ~50× cheaper than HDD/GB
  • Slow access — Mechanical (HDD/tape) or electronic (SSD) but slower than DRAM
  • Secondary role — Stores programs/data not currently in use by CPU

Exam Key Points & Common Questions

Disk Structure
  • Track → Cylinder: all tracks at same radial position across all platters
  • Typical sector size: 512 bytes (modern drives: 4096 bytes)
  • Fixed-head disk: one head per track (fast, expensive, rare)
  • Movable-head disk: one head per surface (common; requires seek)
Performance
  • Sequential access >> random access (up to 440× faster in example)
  • RPS miss adds one full revolution of latency to I/O timeline
  • Average seek time ≠ maximum seek time; non-linear traversal
RAID Key Facts
  • RAID 0: No redundancy — never use alone for critical data
  • RAID 1: Best for write-heavy, highly available, mission-critical data
  • RAID 2: Not commercially used — modern drives have built-in ECC
  • RAID 3: Parallel access only → bad for transaction workloads
  • RAID 4: Parity disk bottleneck → superseded by RAID 5
  • RAID 5: Most popular — distributed parity eliminates bottleneck
  • RAID 6: Best fault tolerance — survives 2 simultaneous failures
SSD
  • Flash erased at block level, not page level → write amplification
  • Wear leveling + TRIM + garbage collection extend SSD lifespan
  • SSD: 10× faster random access than HDD; still more expensive/GB
Optical / Tape
  • CD/DVD use CLV; HDD uses CAV (or MZR)
  • Blu-ray beats HD-DVD; 405-nm blue-violet laser; 25 GB/layer
  • Tape = sequential access only; serpentine recording
  • LTO dominates tape market; capacity doubles every ~2 years
